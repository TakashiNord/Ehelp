1 Клиент сигнальной системы

1.1 Назначение программы

Клиент сигнальной системы (CSignal.exe) предназначен для оперативного[1] и ретроспективного представления зафиксированных в комплексе РСДУ5 событий на клиентском рабочем месте (диспетчера, администратора и т.п.).

1.2 Запуск программы

Как правило, запуск Клиента сигнальной системы осуществляется автоматически, при запуске Панели оператора.

Если клиент сигнальной системы запущен, то на панели приложений рабочего стола должна находиться кнопка с иконкой (Рисунок 1).

Рисунок 1 – Иконка приложения на панели приложений рабочего стола

Если программа автоматически не запустилась, то ее можно открыть следующими способами:

1.    Нажать кнопку «Прикладные программы» на Панели оператора и выбрать из списка «Клиент сигнальной системы». Если данный пункт в меню отсутствует, обратитесь к администратору.

2.    Через ярлык на рабочем столе (если такой имеется).

Примечание. Приложение поддерживает возможность работы в терминальном режиме в среде ОС Windows Server.

 

1.3 Описание основного окна приложения

Вид основного окна клиента сигнальной системы представлен на Рисунок 2.

 

 

 


Рисунок 2 – Основное окно приложения

Главная панель инструментов приложения состоит из следующих компонентов:

ü  выпадающий список  содержит варианты проигрывания звуковых оповещений на клиентском рабочем месте;

ü  кнопка «Прослушать сообщение»  позволяет повторно прослушать полученное сообщение о принятом сигнале;

ü  кнопка «Показывать всплывающие окна»  имеет 2 состояния: «нажата» и «не нажата», и позволяет включить/отключить отображение всплывающих окон с новым сигналом;

ü  кнопка «Следовать за новыми сигналами»  позволяет включить режим отображения оперативных сигналов, когда курсор автоматически переходит на строку с новым сигналом;

ü  кнопка «Очистить»  позволяет выполнить очистку сигналов в таблице «Оперативный режим»;

 

ü  кнопка «Квитировать все»  предназначена для группового квитирования всех неквитированных сигналов на всех открытых в текущий момент схемах/панелях и других визуальных формах АРМ. Кнопка активна, если в таблице «Оперативный режим» имеются сигналы, требующие квитирования.

Квитирование – это подтверждение факта осмысленного восприятия пользователем информации о произошедшем событии в системе.

На схемах/панелях и других визуальных формах изменение текущего состояния коммутационных аппаратов, выход за границы уставок и контроль нулевых значений аналоговых параметров индицируются статусом привлечения внимания – элемент мигает. Сброс индикации привлечения внимания выполняется квитированием. При этом осуществляется запись в журнал квитирования информации о времени, пользователе и параметрах, которые были квитированы;

ü  кнопка «Свойства»  служит для вызова окна «Свойства»;

ü  кнопка «Экспорт в Excel»  позволяет сохранить текущее представление списка сигналов в файл Excel.

Статусная строка отображает количество новых сигналов: первое значение – общее количество, второе значение – количество аварий, третье значение – количество предупреждений, четвертое значение – количество несквитированных сигналов. В правой части указано время с момента запуска приложения.

Рисунок 3 – Статусная строка

Каждый сигнал, представленный в таблице на Рисунок 2, содержит информацию, которая разделена для удобства на несколько столбцов:

ü  Тип – содержит тип события (авария , предупреждение  или уведомление );

ü  Дата – содержит дату события в формате «день месяц год»;

ü  Время – содержит время события в формате «час:минута:секунда.миллисекунда»;

ü  Сигнал – содержит краткую информацию о событии;

ü  Зона ответственности – содержит описание зоны, от которой пришел сигнал[2];

ü  Источник объект – содержит наименование объекта, для которого произошло данное событие. В зависимости от текущих настроек клиента может содержать полный путь к объекту в иерархической структуре предприятия (см. раздел «Индивидуальная настройка клиента»). Для некоторых событий данная информация может отсутствовать;

ü  Источник параметр – содержит наименование параметра системы, который является источником данного события. Для некоторых событий данная информация может отсутствовать;

ü  Сервис – содержит наименование сервиса системы, который является источником данного события (для некоторых событий данная информация может отсутствовать).

1.4 Работа с клиентом сигнальной системы

Основное назначение клиента сигнальной системы – отображение событий по мере их возникновения в системе. Кроме того, в имеющемся ретроспективном режиме доступен просмотр событий из архивных журналов системы за произвольный интервал дат.

Настройки клиента сигнальной системы на обработку сигналов индивидуальны для каждого пользователя, то есть пользователю будут доступны для отображения только те сигналы, которые соответствуют его зоне ответственности[3].

Главное окно клиента сигнальной системы содержит 3 вкладки: «Оперативный режим», «Режим квитирования», «Ретроспективный режим».

1.4.1 Оперативный режим

В оперативном режиме отображаются сигналы, которые поступили в систему с момента запуска клиента сигнальной системы.

В оперативном режиме отображённый сигнал может требовать квитирования. В этом случае он будет непрерывно мигать (периодически меняется цвет фона). Для квитирования сигнала (и остановки мигания) нужно дважды щелкнуть на соответствующем сигнале в списке или выбрать пункт «Квитировать» в контекстном меню сигнала. Если требуется квитировать все сигналы разом, то необходимо нажать кнопку «Квитировать все» на панели инструментов приложения.

1.4.2 Режим квитирования

В режиме квитирования отображаются сигналы, которые поступили в систему с момента запуска приложения и требуют квитирования или были сквитированы. По умолчанию, в таблице отображаются только несквитированные сигналы (активирован фильтр по столбцу «Сквитирован» со значением «Нет»).

Таблица, кроме столбцов, описанных в разделе «Описание основного окна приложения», содержит дополнительно столбцы:

ü  Сквитирован – содержит информацию, был ли сигнал сквитирован;

ü  Время квитирования – содержит время квитирования сигнала.

Квитирование сигнала осуществляется двойным нажатием на соответствующем сигнале или выбором пункта «Квитировать» в контекстном меню сигнала.

1.4.3 Ретроспективный режим

В ретроспективном режиме пользователь может просматривать сигналы за любую дату. Для этого пользователю необходимо указать интервал дат и нажать кнопку «Показать».

Рисунок 4 – Ретроспективный режим

Во время загрузки сигналов в ретроспективном режиме есть возможность отмены загрузки сигналов. Кнопка «Отмена» отображается только во время загрузки сигналов (Рисунок 5). При этом в таблице будет отображено то количество сигналов, которое успело отобразиться в таблице до нажатия отмены.

Рисунок 5 – Загрузка сигналов

Примечание. После запуска клиента сигнальной системы во вкладку ретроспективного режима загружаются сигналы, которые поступили в систему с начала текущих суток до момента запуска клиента сигнальной системы. Далее загрузка сигналов происходит по инициативе пользователя с помощью кнопки «Показать».

1.4.4 Озвучивание сигналов

Во время поступления события клиент сигнальной системы выполняет его озвучку, проговаривая тип сигнала, объект источник и параметр источник[4]. Если сигналу назначен дополнительный звуковой сигнал[5], то он будет воспроизведён перед основным озвучиванием.

При большом количестве одновременно пришедших сигналов могут быть озвучены не все сигналы; предпочтение отдается новым.

Рисунок 6 –Варианты проигрывания звуковых оповещений

Для управления звуком приложения пользователь может использовать варианты из выпадающего списка на панели инструментов приложения (Рисунок 6). Для выбора доступны три положения: воспроизводить звук всегда, воспроизводить звук только при аварии, воспроизведение звука отключено.

Дополнительные настройки озвучивания сигналов описаны в разделе «Индивидуальная настройка клиента».

1.4.5 Повторное прослушивание сообщений

В оперативном и ретроспективных режимах, а также режиме квитирования доступна функция повторного прослушивания сообщений о полученных сигналах.

Приложение позволяет воспроизвести звуковые дорожки, связанные с сигналом, в любой момент по требованию пользователя.

Приоритет воспроизведения сигналов отдан в пользу новых сигналов, полученных в оперативном режиме.

Воспроизведение сигнала можно осуществить одним из способов:

ü  с помощью кнопки «Прослушать сообщение» на панели инструментов;

ü  из контекстного меню сигнала – пункт «Прослушать сообщение».

Рисунок 7 – Повторное прослушивание сообщения

Примечание. Пункт меню «Прослушать сообщение» и кнопка «Прослушать сообщение» доступны только для полностью озвученных сигналов в случае, если активирована настройка «Проигрывать только полностью озвученные сигналы» (подробная информация в разделе «Индивидуальная настройка клиента»).

1.4.6 Отображение всплывающих сообщений

Существуют сигналы, требующие оповещения пользователя в виде всплывающего окна.

Рисунок 8 – Отображение всплывающего окна

Всплывающее окно содержит информацию о сигнале и отображается поверх всех остальных окон системы. Если нажать на кнопку «Показать окно сигналов», пользователь увидит основное окно клиента сигнальной системы с открытой вкладкой оперативного режима, само всплывающее окно при этом исчезнет. Кнопка «Закрыть» закроет всплывающее окно.

1.4.7 Переход на схему при получении сигнала от параметра

При получении предупреждающего сигнала или сигнала об аварии от параметра режима, имеющего настройку на схему, можно осуществить переход на эту схему непосредственно из клиента сигнальной системы. Аналогичным образом осуществляется переход при получении сигнала от коммутационного аппарата или устройства СРЗиСА.

Для перехода к параметру на схеме необходимо вызвать контекстное меню на строке с сигналом и в меню выбрать схему для перехода. После осуществления выбора открывается соответствующая схема, и устанавливается фокус на параметр, по которому был сигнал.

Рисунок 9 – Переход на схему

Таким образом можно осуществить переход на схему как из оперативного и ретроспективного режима просмотра, так и из режима квитирования.

Кроме того, такой переход доступен из всплывающего сообщения. Для этого нужно вызвать контекстное меню в поле описания свойств сигнала. Контекстное меню имеет тот же вид, что и при переходе с ретроспективного/оперативного режима.

На время перехода из клиента сигнальной системы к активному элементу на схеме появится блокирующее окно с надписью (Рисунок 10). Для отмены открытия схемы можно нажать клавишу Esc.

1.5 Индивидуальная настройка клиента

1.5.1 Главное меню

Главное меню в клиенте сигнальной системы содержит три раздела.

Рисунок 11 – Главное меню

1.5.1.1 Меню «Файл»

В разделе «Файл» определены команды:

ü  «Экспорт в Excel» – сохраняет текущее содержимое таблицы с сигналами в файл формата EXCEL;

ü  «Выход» – завершает работу с приложением.

1.5.1.2 Меню «Вид»

В пункте «Вид» расположены следующие команды для изменения параметров отображения сигналов:

ü  «Очистить» – очищает таблицу с сигналами в оперативном режиме;

ü  «Строка состояния» – включает/выключает отображение строки состояния. В этой строке отображается информация о работе программы и общее время, прошедшее с начала запуска приложения;

ü  «Поверх всех окон» – включает/выключает отображение окна клиента сигнальной системы поверх других окон системы;

ü  «Свойства…» – вызывает окно основных настроек приложения (Рисунок 12)[6].

Рисунок 12 – Вкладка «Общие» окна «Свойства»

В окне «Свойства» при выборе вкладки «Общие» выбираются следующие опции настройки (Рисунок 12):

ü  «Размер шрифта сообщений» – устанавливает размер шрифта в названиях кнопок главного меню и панели инструментов, заголовках окон и информационных сообщениях в таблице сигналов;

ü  «Индикация на панели задач при получении сигнала с типом «Авария» – включает/отключает индикацию на панели задач при поступлении сигнала типа «Авария»;

ü  «Открывать схемы всегда в новом окне» – устанавливает правило открытия всех схем в собственных экземплярах приложения «Просмотр схем» (новых окнах);

ü  «Показывать развернутое имя объекта-источника» – позволяет отображать имя объекта-источника в окне списка сигналов в соответствии с его иерархическим описанием в информационной модели[7];

ü  «Обрабатывать сигналы от неизвестных объектов» – позволяет отображать сигналы с объектом источником, который не зарегистрирован в системе.

ü  «Проигрывать только полностью озвученные сигналы» – разрешает воспроизводить озвучивание сигналов, у которых определены звуковые файлы для объекта-источника и параметра-источника, а также активировано свойство сигнала «Проговаривание»[8].

Если хотя бы один из перечисленных параметров поступившего сигнала не задан, то он не будет озвучен клиентом сигнальной системы.

ü  «Не отображать миллисекунды меток времени сигнала» – скрывает значение миллисекунд в колонке «Время» таблицы сигналов.

Для удобства просмотра сигналов в «Оперативном режиме» настраиваются значения следующих параметров:

ü  максимальное количество отображаемых сигналов (от 1 до 1000). При достижении максимального количества сигналов удаляются старые сигналы и добавляются новые;

ü  максимальное количество дней для отображения сигналов (от 1 до 100). В клиенте сигнальной системы отображаются только сигналы с датой, равной текущей дате, и «Максимальное количество дней» до нее;

ü  максимальное количество сигналов во всплывающем окне (от 1 до 50). При поступлении количества сигналов, превышающих максимальное количество сигналов во всплывающем окне, самый старый сигнал удаляется, в верх списка добавляется новый сигнал.

Для настройки сигнала привлечения внимания:

ü  для активации сигнала привлечения внимания необходимо установить галочку напротив пункта «Включить»;

ü  для установки интервала повтора сигнала привлечения внимания следует ввести необходимое значение в соответствующее числовое поле.

Действия для повторов сигналов, которые были сквитированы:

ü  если сигнал был сквитирован пользователем, то при повторном поступлении сигнала можно отключить повторное озвучивание сигнала (установить галочку напротив пункта «Не озвучивать»), отключить индикацию сигнала в списке сигналов (установить галочку напротив пункта «Не мигать») и отключить отображение во всплывающем окне (установить галочку напротив пункта «Не отображать во всплывающем окне»).

На вкладке «Настройка предпочтений для сигналов» отображается список поддерживаемых сигналов в иерархии зон ответственности и групп сигналов (Рисунок 13). Галочками выделяются те сигналы, которые может принимать и обрабатывать текущий пользователь. Этот выбор осуществляется по согласованию с администратором комплекса.

Примечание. При отсутствии у пользователя права «Настройка для себя обработки сигналов» вкладка «Настройка предпочтений для сигналов» не отображается.

1.5.3 Фильтрация и группировка сигналов

В ретроспективном, квитирования и оперативном режиме доступна функция фильтрации по любой колонке списка сигналов.

Для того чтобы вызвать фильтр, нужно кликнуть указателем мыши на кнопку фильтра (кнопка изображена в виде воронки) в правой части заголовка колонки.

Фильтр может быть задан двумя способами: фильтр по маске, фильтр по значениям. Данные фильтры применяются одновременно.

1.5.3.1 Фильтр по маске

Фильтр по маске представляет собой поле ввода, располагающееся в верхней части фильтра. Для этого необходимо нажать правой кнопкой в таблице сигналов и выбрать пункт «Показать строку автофильтра». Если ввести в поле ввода фильтра текст, в списке сигналов останутся только те сигналы, у которых в значениях данной колонки присутствует этот текст (Рисунок 15).

Рисунок 15 – Фильтр по маске

1.5.3.2 Фильтр по значениям

Фильтр по значениям представляет собой список всех значений, присутствующих в данной колонке (Рисунок 16). Разрешенные значения выделены галочками. Пользовать может включать или выключать значения в колонке. В первой строке списка находится значение «Показать все». Если установить галочку напротив него, будут выделены все значения.

Рисунок 16 – Фильтр по значениям

Фильтры по колонкам суммируются. В списке сигналов остаются только те сигналы, которые удовлетворяют фильтрам всех колонок. Серая воронка, расположенная левее кнопки фильтра, указывает на то, что для данной колонки задан какой-либо фильтр.

1.5.3.3 Конструктор фильтра

Конструктор фильтра позволяет отфильтровать сигналы в таблице, соответствующие заданным сложным условиям поиска. Вызов конструктора осуществляется через контекстное меню таблицы – пункт «Конструктор фильтра».

Рисунок 17 – Окно конструктора фильтра

В появившемся окне необходимо задать одно или несколько условий поиска (с использованием логических операций «И», «ИЛИ», «НЕ И» и «НЕ ИЛИ») для одного или нескольких столбцов таблицы.

Щелчок левой кнопкой мыши по названию логической операции позволяет изменить текущую операцию на другую, добавить условие или группу условий в выражение или очистить условия фильтра (Рисунок 18).

Рисунок 18 – Выбор логической операции

Щелчок левой кнопкой мыши по названию столбца таблицы позволяет выбрать новый столбец и заменить им текущий столбец (Рисунок 19).

Рисунок 19 – Выбор столбца

Условие сравнения результатов поиска с искомым значением задается аналогично вышеописанным функциям.

Рисунок 20 – Выбор искомого значения

Для завершения конструирования сложного условия поиска записей в таблице и осуществления поиска записей, удовлетворяющих выражению, необходимо нажать кнопку «Применить».

 

1.5.3.4 Группировка

Для группировки сигналов необходимо правой кнопкой мыши нажать на заголовок столбца и выбрать из контекстного меню пункт «Группировать по этой колонке» (Рисунок 21).

Рисунок 21 – Группировка сигналов

Поддерживается группировка по нескольким параметрам. Уровни группировки можно регулировать.

Для того чтобы сбросить группировку, в контекстном меню необходимо выбрать пункт «Разгруппировать» либо вручную перенести название столбцов на место в таблице.

Примечание. Группировка по столбцу «Время» не поддерживается, однако можно выполнить группировку по дате.

1.5.4 Изменение видимости колонок

Изменение видимости колонок осуществляется из контекстного меню заголовка таблицы (Рисунок 22). Вызов контекстного меню осуществляется нажатием правой кнопкой мыши по заголовку. Подробное описание работы с контекстным меню доступно в документе «Контекстное меню приложений РСДУ. Руководство пользователя».

Рисунок 22 – Контекстное меню таблицы сигналов

Для того чтобы скрыть колонку необходимо выбрать пункт «Скрыть колонку» контекстного меню либо потянуть название колонки вниз.

Примечание. Для колонок «Тип», «Дата», «Время» и «Сигнал» функция скрытия недоступна.

Для того чтобы отобразить скрытые колонки необходимо выбрать пункт «Выбор колонок» контекстного меню. Появится окно со списком скрытых колонок (Рисунок 23). Необходимо перенести колонку в требуемую позицию нажатием и удержанием левой кнопкой мыши или дважды нажать левой кнопкой мыши на колонку в списке. В последнем варианте колонка будет установлена в положении «По умолчанию».

Рисунок 23 – Список скрытых колонок

Порядок отображения и ширина колонок сохраняются в процессе работы приложения, а также при его закрытии.

1.6 Завершение работы приложения

Для выхода из Клиента сигнальной системы необходимо выбрать в меню «Файл» команду «Выход» или нажать на крестик в правой верхней части окна приложения.

Предупреждение. Выход из приложения возможен в том случае, если у пользователя есть права на управление клиентом сигнальной системы. Иначе выход из Клиента сигнальной системы будет возможен только с закрытием приложения «Панель оператора».
