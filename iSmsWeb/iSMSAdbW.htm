Установка WEB-сервера

Установка системного программного обеспечения

1.        Обновление репозитория перед началом установки.

Обновление репозитория выполняется с помощью команды:

sudo apt-get update

Обновление пакетов выполняется с помощью команды:

sudo apt-get upgrade

2.        Установка веб-сервера apache.

Если сервер apache уже установлен:

ü завершить процесс (если запущен):

sudo /etc/init.d/apache2 stop

ü удалить apache:

sudo apt-get --purge remove apache2

ü удалить конфигурационные файлы из папки /etc/apache/:

rm -r /etc/apache/

ü установить с ключом --reinstall:

sudo apt-get install --reinstall apache2

Если apache не был ранее установлен:

ü выполнить установку с помощью команды:

sudo apt-get install apache2

После установки запустите сервер apache с помощью команды:

sudo /etc/init.d/apache2 start

Если установка прошла успешно, то после ввода в браузер IP-адреса машины, на которую производилась установка (например, http://192.168.20.156/), отобразится страница «It works!»

3.        Включение модуля mod_rewrite, необходимого для работы WEB-сервера.

Выполните команду:

sudo a2enmod rewrite

Чтобы изменения вступили в силу, перезапустите сервер apache с помощью команды:

sudo /etc/init.d/apache2 restart

4.        Установка php.

Если php уже был ранее установлен, необходимо выполнить следующие шаги:

ü  удалить php:

sudo apt-get --purge remove php5 && sudo apt-get --purge remove php5-common

ü  удалить конфигурационные файлы из папки /etc/php5/:

rm -r /etc/php5/

ü  установить с ключом --reinstall:

sudo apt-get install --reinstall php5 libapache2-mod-php5 php5-common php5-cli php5-mysql

Если php не был ранее установлен, выполнить установку с помощью команды:

ü  sudo apt-get install php5 libapache2-mod-php5 php5-common php5-cli php5-mysql

Установка Oracle Client

Клиент Oracle доступен для скачивания с официального сайта.

1.      Зайдите на сайт по представленной ссылке, выберите систему/архитектуру и скачайте rpm-пакеты с названиями basic, devel (оно же sdk), sqlplus.

2.  Установка в Debian/Ubuntu выполняется с помощью утилиты alien

alien -i oracle-instantclient11.2-basic-12.1.0.2.0-1.x86_64.rpm

alien -i oracle-instantclient11.2-devel-12.1.0.2.0-1.x86_64.rpm

alien -i oracle-instantclient11.2-sqlplus-12.1.0.2.0-1.x86_64.rpm

3.      Программа будет установлена в каталоги /usr/lib/oracle/12.1/client64 и /usr/include/oracle/12.1/client64.

Необходимо также создать дополнительную ссылку:

ln -s /usr/include/oracle/12.1/client64 /usr/lib/oracle/11.2/client64/include

4.  Для ldconfig создаём конфиг /etc/ld.so.conf.d/oracle.conf со следующим содержимым:

/usr/lib/oracle/12.1/client64/lib

5.  Устанавливаем переменную окружения ORACLE_HOME с помощью нового конфига /etc/profile.d/oracle.sh со следующим содержимым:

export ORACLE_HOME=/usr/lib/oracle/12.1/client64

Эту же строчку можно добавить в пользовательском конфигурационном файле ~/.bashrc

6.  Обновление загрузчика библиотек с помощью команды:

ldconfig

7.  Проверка работы клиента с помощью команды

sqlplus64 <login>/<password>@<host>:<port>/<dbname>

Сборка PHP с поддержкой oracle

1.      Перед сборкой php необходимо удовлетворить некоторые зависимости: apxs2 и libxml.

apt-get update
 
#Поиск пакета, содержащего apxs2
apt-cache search apxs2
 
#Установка пакета, содержащего apxs2
apt-get install apache2-dev
 
#Поиск пакета, содержащего libxml
apt-cache search libxml
 
#Установка пакета, содержащего libxml
apt-get install libxml++2.6-dev

2.      Далее качаем исходники php с официального сайта

3.      Распаковываем в папку для сборки

4.      Настраиваем сборку и компилируем:

./buildconf --force
./configure \
   --prefix=/usr \
   --with-apxs2=/usr/bin/apxs2 \
   --with-zlib-dir=/usr/lib \
   --with-mysql=/usr \
   --with-pdo-mysql=/usr \
   --enable-soap \
   --enable-mbstring \

   --enable-zend-multibyte \

   --with-oci8=instantclient,/usr/lib/oracle/12.1/client64/lib \
   --with-pdo-oci=instantclient,/usr,12.1
make
checkinstall

Если сборка будет выполняться несколько раз, то перед последующим выполнением команды ./configure необходимо будет выполнять make distclean.

Если ./configure будет ругаться на не поддерживаемую версию oracle, то найдите в файле configure примерно следующие строчки:

case $PDO_OCI_VERSION in
   9.0|10.1|10.2|11.1|11.2)

и допишите вашу версию клиента.

Настройка Apache

При выполнении команды ./configure --prefix=/usr конфиги apache будут полностью аналогичны идущим со сборкой php из официальных пакетов. На всякий случай они приведены ниже

/etc/apache2/mods-available/php5.load:

LoadModule php5_module /usr/lib/apache2/modules/libphp5.so

/etc/apache2/mods-available/php5.conf:

<FilesMatch ".+\.ph(p[345]?|t|tml)$">
    SetHandler application/x-httpd-php
</FilesMatch>
<FilesMatch ".+\.phps$">
    SetHandler application/x-httpd-php-source
    # Deny access to raw php sources by default
    # To re-enable it's recommended to enable access to the files
    # only in specific virtual host or directory
    Order Deny,Allow
    Deny from all
</FilesMatch>
# Deny access to files without filename (e.g. '.php')
<FilesMatch "^\.ph(p[345]?|t|tml|ps)$">
    Order Deny,Allow
    Deny from all
</FilesMatch>
 
# Running PHP scripts in user directories is disabled by default
# 
# To re-enable PHP in user directories comment the following lines
# (from <IfModule ...> to </IfModule>.) Do NOT set it to On as it
# prevents .htaccess files from disabling it.
<IfModule mod_userdir.c>
    <Directory /home/*/public_html>
        php_admin_value engine Off
    </Directory>
</IfModule>
 
Для корректного отображения иконок в WEB-приложении iSMS необходимо запретить использование алиаса /icons/ в конфигурационном файле сервера Apache:
/etc/apache2/mods-available/alias.conf
Для этого в файле нужно закомментировать соответствующую строку (с помощью символа #), например, как указано ниже:
#Alias /icons/ "/usr/share/apache2/icons/"

Проверка конфигурации интерпретатора php и веб-сервера apache

1.      Проверьте наличие раздела "oci8" в phpinfo

2.      Проверка соединения с помощью официального клиента:

sqlplus64 <login>/<password>@<host>:<port>/<dbname>

3.      Проверку соединения с помощью php можно сделать небольшим скриптом, например, таким: 

<?php
 
$tns = "(DESCRIPTION = (ADDRESS = (PROTOCOL = TCP)(HOST = <host>)(PORT = <port>)) (CONNECT_DATA = (SERVER = DEDICATED) (SERVICE_NAME = <dbname>) ) )";
$user = "...";
$password = "...";
try {
    $pdo = new \PDO("oci:dbname=" . $tns . ";charset=utf8", $user, $password);
    var_dump($pdo);
}
catch (\Exception $e) {
    echo ($e->getMessage());
}

Конфигурирование виртуального хоста в apache

1          Убедитесь, что сервер apache работает от пользователя www-data. Иначе, нужно указать, чтобы он работал от имени www-data. Для этого в одном из конфигурационных файлов веб-сервера apache (/etc/apache2/envvars) нужно найти следующие директивы:

APACHE_RUN_USER=www-data

APACHE_RUN_GROUP=www-data

2          Создайте папку /home/www-data/www/htdocs/WASUTP

# mkdir -p /home/www-data/www/htdocs/WASUTP

3          Скопируйте в директорию /home/www-data/www/htdocs/WASUTP архив wasutp.tar.gz

4          Распакуйте содержимое архива wasutp.tar.gz

# cd /home/www-data/www/htdocs/WASUTP

# tar -xvf ./wasutp.tar.gz

5          Установить владельца www-data для дистрибутива:

# chown -R www-data:www-data /home/www-data

6          Настроить DNS для домена, который будет ссылаться на ip-адрес веб-сервера (например rsdu2web), если WASUTP должен быть доступен по домену. Для тестового режима можно обращаться к веб-серверу по ip из браузера, либо прописать строку в файле /etc/hosts/ (в ОС семейства линукс) или C:\Windows\System32\drivers\etc\hosts (в ОС семейства windows) на машине клиента. Например: 192.168.20.156  rsdu2web www.rsdu2web

7          Конфигурация виртуального хоста в Debian 6.0

7.1    Отключить виртуальный хост «по умолчанию»

# a2dissite default

 

Site default disabled.

Run '/etc/init.d/apache2 reload' to activate new configuration!

7.2    Создать файл /etc/apache2/sites-available/wasutp с описанием виртуального хоста WASUTP.

# touch /etc/apache2/sites-available/wasutp

7.3    Отредактировать файл /etc/apache2/sites-available/wasutp. Описать виртуальный хост. Например:

<VirtualHost *:80>

         ServerName rsdu2web

         ServerAlias www.rsdu2web

         DocumentRoot /home/www-data/www/htdocs/WASUTP/public

         <Directory /home/www-data/www/htdocs/WASUTP/public>

                         AllowOverride All

                         Options +FollowSymLinks

                         Order Deny,Allow

                         Allow from all

         </Directory>

</VirtualHost>

7.4    Включить сайт для apache2:

# a2ensite wasutp

7.5    Чтобы изменения вступили в силу необходимо перезапустить apache:

# /etc/init.d/apache2 restart

Структура каталогов приложения

WEB-сервер iSMS имеет определенную структуру каталогов, изменение которой может привести к неработоспособности системы.

Описание каталогов приложения

сonfig

каталог, содержащий основные файлы конфигурации приложения, в которых описываются: параметры подключения к субд, путь к утилите rtquery,  периоды обновления схем и информационных панелей, режимы работы схем и информационных панелей (тестовый или нормальный), заголовок окна браузера, параметр «replaceIp» и другие.

language

каталог, содержащий файлы локализации приложения zend framework.

module

каталог, содержащий исполняемые файлы модулей приложения zend framework.

public

каталог, содержащий файлы исходного кода приложения Ext JS и другие статические файлы с расширениями js, css и изображения, а также файл .htaccess содержащий данные для переопределения конфигурации веб-сервера и индексный файл index.php, на который перенаправляются все запросы пользователей, с помощью .htaccess.

public/resource – каталог, содержащий файлы ресурсов приложения:

            reports - конфигурационные файлы отчетов,

            panels - конфигурационные файлы информационных панелей,

            schemes - конфигурационные файлы схем.

tmp

каталог, содержащий конфигурационные файлы для утилиты rtquery, а также временные файлы, создаваемые приложением.

vendor

каталог, содержащий библиотеку классов приложения и определения системы автозагрузки классов.

Дополнительная конфигурация WEB-сервера iSMS

Настройка операционной системы

Если веб-клиент запускается в браузере операционной системы семейства Windows, убедитесь, что в настройках системы отключен режим гибернации, поскольку, из-за специфичности этого режима, веб-клиент в нем корректно работать не будет.

 

Установка прав для веб-сервера apache

Веб-клиент использует права root для некоторых задач (перезагрузка веб-сервера, перезагрузка модулей ema, выключение сервера, чтение логов и т.д.), поэтому необходимо добавить пользователя, от которого работает веб-сервер apache (по умолчанию www-data) в группу sudoers и разрешить выполнять sudo - команды без запроса пароля, если это не было выполнено ранее.

Порядок действий:

1.        Добавить пользователя в группу sudo

# usermod -a -G sudo www-data

2.        Проверка групп, в которые входит пользователь

# id www-data

3.        Разрешить sudo-пользователям выполнение команд sudo без ввода пароля. Для этого необходимо открыть файл /etc/sudoers, найти в нем строку, начинающуюся с %sudo и заменить ее на %sudo ALL=NOPASSWD: ALL

Настройка сессий php

По умолчанию веб-сервер хранит информацию о сессиях в файлах сессий, каждый из которых хранится в течении «времени жизни сессий». По умолчанию это время соответствует 1440 или 1800 секундам. Чтобы сессия не истекла во время бездействия веб-клиента необходимо найти директиву session.gc_maxlifetime в файле php.ini и изменить ее значение на 604800 (session.gc_maxlifetime = 604800), после чего, чтобы изменения вступили в силу, перезагрузить сервер, для чего необходимо выполнить в командной строке:

# service apache2 restart

Стандартное расположение файла php.ini - /etc/php5/apache2/php.ini. Если файл php.ini отсутствует в указанном каталоге, точное его расположение можно узнать, выполнив в php комманду «echo phpinfo()». Для этого в файле /home/www-data/www/htdocs/WASUTP/public/index.php нужно раскомментировать строку

//echo phpinfo();die();

(убрать символы //), сохранить файл, запустить веб-клиент в окне браузера и найти там строку «Loaded Configuration File»

 

После того, как вы нашли путь к файлу php.ini, закомментируйте строку обратно и обновите страницу браузера.

Настройка ssl-шифрования (опционально)

1.      Создание сертификата

Здесь приведен пример создания самоподписанного сертификата ssl, – такой сертификат бесплатен. При первом обращении к серверу веб браузер выведет предупреждение о том, что «Сертификат безопасности сайта не является доверенным», и попросит добавить исключение безопасности. Использование доверенного сертификата не является бесплатным и выходит за рамки документации.

Установка OpenSSL

# apt-get update

# apt-get install openssl

Создание сертификата

# openssl req -new -x509 -nodes -out server.crt -keyout server.key

После создания, файлы server.crt и server.key следует перенести в каталог /etc/apache2/ssl/ (если он отсутствует – создать его). Путь к ним будет указан далее, в время настройки виртуального хоста.

2.      Включение модуля mod_ssl веб-сервера apache

# a2enmod ssl

         Модуль будет включен после перезагрузки веб-сервера:

# service apache2 restart

либо

# /etc/init.d/apache2 restart

3.        Настройка виртуального хоста веб-сервера

Необходимо в конфигурационном файле, в котором описан виртуальный хост apache (/etc/apache2/sites-available/wasutp) создать схожую конфигурацию, использующую ssl для 443-го порта (порт https по умолчанию) , для этого нужно дописать в конец файла:

<VirtualHost *:443>

         ServerName rsdu2web

         ServerAlias www.rsdu2web

         DocumentRoot /home/www-data/www/htdocs/WASUTP/public

         <Directory /home/www-data/www/htdocs/WASUTP/public>

                         AllowOverride All

                         Options +FollowSymLinks

                         Order Deny,Allow

                         Allow from all

 

                               SSLEngine on

                               SSLCertificateKeyFile /etc/apache2/ssl/server.key

                               SSLCertificateFile    /etc/apache2/ssl/server.crt

 

         </Directory>

</VirtualHost>

Чтобы настройки вступили в силу необходимо перезагрузить веб сервер.

4.        Настройка 301 редиректа

Для того, чтобы все соединения по незащищенному протоколу http перенаправлялись на https необходимо раскомментировать  в файле (либо дописать в конец файла) /home/www-data/www/htdocs/WASUTP/public/.htaccess строки

RewriteCond %{SERVER_PORT} 80

RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]

Обновление WEB-сервера

Обновление программного обеспечения

1.      Сделайте резервную копию рабочего каталога виртуального хоста (путь по умолчанию /home/www-data/www/htdocs/WASUTP).

Например,

# mv /home/www-data/www/htdocs/WASUTP /home/www-data/www/htdocs/WASUTP2

2.      Сделайте резервную копию БД iSMS

3.      Распакуйте дистрибутивы программного обеспечения в каталог, на который настроен виртуальный хост веб-сервера.

ü  Zip-архив

# cd /home/www-data/www/htdocs/

# cp путь_к_файлу_дистрибутива wasutp.zip

# unzip wasutp.zip

ü  Tar.gz-архив

# cd /home/www-data/www/htdocs/

# cp путь_к_файлу_дистрибутива wasutp.tar.gz

# tar xvfz wasutp.tar.gz

ü  Tar.bz2-архив

# cd /home/www-data/www/htdocs/

# cp путь_к_файлу_дистрибутива wasutp.tar.bz2

# tar xvjf wasutp.tar.bz2

В результате должна получиться структура каталогов:

/home/www-data/www/htdocs/WASUTP/config/…

/home/www-data/www/htdocs/WASUTP/data/…

/home/www-data/www/htdocs/WASUTP/languag/…

/home/www-data/www/htdocs/WASUTP/module/…

/home/www-data/www/htdocs/WASUTP/public/…

/home/www-data/www/htdocs/WASUTP/tmp/…

/home/www-data/www/htdocs/WASUTP/vendor/…

4.      Создайте файл wasutp.config.php с конфигурационными данными (можно переместить из резервной копии)

# cd /home/www-data/www/htdocs/WASUTP

# cp ../WASUTP2/config/wasutp.config.php config/wasutp.config.php

После переноса файла wasutp.config.php из бэкапа проверьте в нем наличие всех параметров (некоторых параметров, возможно, не было в предыдущей версии веб-клиента). Список параметров есть в разделе «Установка Web-сервера/ Конфигурирование WEB-сервера iSMS»

5.      Скопируйте имеющиеся файлы ресурсов для информационных панелей, схем и отчетов в каталоги WASUTP/public/resource/panels, WASUTP/public/resource/schemes и WASUTP/public/resource/reports

# cp -r путь_к_папке_с_панелями/* /home/www-data/www/htdocs/WASUTP/public/resource/panels/

# cp -r путь_к_папке_со_схемами/* /home/www-data/www/htdocs/WASUTP/public/resource/schemes/

# cp -r путь_к_папке_с_отчетами/* /home/www-data/www/htdocs/WASUTP/public/resource/reports/

6.      Установите на папку /home/www-data/www/htdocs/WASUTP владельца www-data и группу www-data

# chown -R www-data:www-data /home/www-data/www/htdocs/WASUTP

7.      Добавьте права на запись в папку WASUTP/tmp и WASUTP/public/resource

# cd /home/www-data/www/htdocs/WASUTP

# chmod -R 777 tmp

# chmod -R 777 public/resource

 

8.      Выполните обновление структуры базы данных, если таковое требуется

9.      В новой версии веб-клиент использует права root для некоторых задач (перезагрузка веб-сервера, перезагрузка модулей ema, выключение сервера, чтение логов и т.д.), поэтому необходимо добавить пользователя, от которого работает веб-сервер apache (по умолчанию www-data) в группу sudo и разрешить выполнять sudo – команды без запроса пароля, если это не было проделано ранее. Действия, направленные на выполнение этой задачи описаны в разделе «Дополнительная конфигурация WEB-сервера iSMS/Установка прав для веб-сервера apache»
