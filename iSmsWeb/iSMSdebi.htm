Аннотация

Данное руководство содержит информацию по установке операционной системы Linux для комплексного решения для АСУТП подстанции – сбор и передача телеинформации с устройств нижнего уровня и предоставление оперативной информации в виде информационных панелей и структурных схем.

Конфигурация оборудования

ü Промышленный компьютер (MOXA DA-682 или аналогичный с объёмом системного диска не менее 1Гб).

ü Дополнительная Flash карта объемом не менее 16GB для хранения БД и архивов.

Необходимые условия

ü Загрузочный диск LiveCD (Hiren’s boot CD или любой Linux-based LiveCD) при установке из готового образа системы.

ü Дистрибутив операционной системы: Debian GNU/Linux 7.4+.

ü USB-DVD

ü Образ начальной сконфигурированной системы.

ü Дистрибутив с приложениями RTL c конфигурационными файлами.

ü Дистрибутив с приложениями WEB.

ü Схема сети комплекса.

Настройка сервера из готового образа системы

1.    Устанавливаем дополнительную Flash Card во внешний слот CF.

2.    Запускаем сервер.

3.    Устанавливаем в BIOS загрузку с носителя CD-ROM/Flash, на котором расположен загрузчик LiveCD.

3.1    Во время перезагрузки системы жмем кнопку DEL для входа в BIOS сервера

3.2    Проверяем включенную поддержку USB-клавиатуры в BIOS (Peripherals/USB Keyboard support – [Enabled])

3.3    В пункте BIOS Advanced/Hard Disk Boot Priority с помощью кнопок PgUp/PgDown перемещаем обнаруженную USB флешку на первую позицию.

3.4    Две карты CF должны быть в следующем порядке: 2-я – системная (1 Гб), 3-я – для данных (16 Гб).

3.5    Сохраняем текущую конфигурацию с помощью F10 и перегружаемся

4.    Загружаемся с Flash drive

4.1    Hiren’s Boot CD [5]

4.2    Выбираем Linux Based rescue environment (Parted Magic 6.7), далее жмем start

5.    Запускаем терминал (Для hiren’s boot CD – LXTerminal. через главное меню или с панели быстрого запуска.)

6.    Проверяем, какие диски, каким устройствам в системе назначены с помощью команды

# fdisk -l

Например,

·           Системный Flash Drive MOXA объемом 1 ГБ обнаруживается как /dev/sda

·           Дополнительный Flash Drive объемом 16 ГБ обнаруживается как /dev/sdb

·           Загрузочный Flash Drive обнаруживается как /dev/sdc

7.    Устанавливаем подготовленный образ системы из файла moxa.deb6-sda-v2.2.img (имя файла может отличаться, команды ниже тогда необходимо скорректировать соответствующим образом).

7.1    Монтируем флешку

# mkdir /mnt/flash

# mount /dev/sdc1 /mnt/flash

В случае нормального выполнения команды ничего не будет выведено на экран. В случае ошибки будет выведено сообщение об ошибке.

7.2    Копируем образ

# dd if=/mnt/flash/images/moxa.deb6-sda-v2.3.img of=/dev/sda bs=64k

В случае нормального выполнения команды ничего не будет выведено на экран. В случае ошибки будет выведено сообщение об ошибке.

7.3    Проверяем разделы /dev/sda.

Надо убедиться, что на диске sda присутствует только один раздел – sda1.

1.         Меняем права доступа для директории /var/lib/mysql. Устанавливаем владельцем пользователя mysql и устанавливаем права на директорию в значение 700.

# chown mysql:mysql /var/lib/mysql

# chmod 700 /var/lib/mysql

2.         Меняем права доступа для директории /tmp.

# chmod 1777 /tmp
