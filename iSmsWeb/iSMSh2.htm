Конвертация схем Modus → iSMS (RTree)

Документ описывает процесс конвертации схем из Modus в iSMS (RTree) с учётом сохранения всех источников данных, настроенных в РСДУ.

RTree — это основной формат схем для работы в клиенте iSMS. Эти схемы  расположены в каталоге public/resource/schemes на сервере с iSMS с расширением .sch.

Идентификатор схемы

Прежде всего необходимо узнать идентификатор схемы Modus в РСДУ. Эту информацию можно получить в интерфейсе базы данных.

Идентификатор понадобится для копирования источников данных в схему.

Конвертация схем из формата Modus

В среде ОС Windows:

1.      Распаковать архив Xsde2Xml.zip.

2.      Скопировать в каталог Xsde2Xml схемы Modus в формате xsde.

3.      Запустить на исполнение командный файл Xsde2Xml.bat.

В результате будут созданы файлы схем вида <Имя_схемы>.xsde.uni.

В редакторе схем iSMS:

1.      Включить режим разработчика в iSMS.

2.      Подготовить php-скрипты для импорта схем.

3.      Подготовить сами схемы.

Важно: Осуществлять импорт схем в iSMS могут только пользователи с правами «Администратор» и «Администратор БД».

Режим разработчика

Включить режим можно любым из способов, описанных далее.

1. Через конфигурационный файл iSMS. Режим разработчика включается в конфиге iSMS (в php-файле config/ wasutp.config.php):

<?php

return array(

    // ...

    'debugMode' => true,

    // ...

);

2. Через пользовательский интерфейс редактора схем. Необходимо открыть новую схему, в меню «Настройки» (кнопка ) поставить галочку «Режим разработки»:

После установки галочки необходимо полностью обновить страницу браузера с iSMS.

После включения режима разработчика в диалоге открытия схем становится доступна вкладка «Extended», где задаются параметры для импорта.

Подготовка php-скриптов для импорта схем

Архив с php-скриптами: php-scripts-uni.zip

Скачайте архив со скриптами и распакуйте в каталог public/php. Скрипты настроены именно на такое размещение. Если размещаете их в другом каталоге, то необходимо будет изменить пути в common.php (см. константы SCHEMES_DIR и SCHEMES_RELATIVE_DIR).

Подготовка схем uni

Поместите импортируемые схемы (SchemeName.xsde.uni) в каталог public/ resource/schemes.

Импорт схем

1.   Создать новую схему: «iSMS»/«Схемы объектов»/«Новая ...».

Она откроется в режиме редактирования.

2.   В меню кнопки  выбрать пункт «Открыть».

Появится диалог открытия схемы.

3.   Во вкладке «Extended» обязательно следует заполнить следующие поля:

ü  Формат: «uni» (выбрать из выпадающего списка)

ü  URL getScheme: «/php/load_scheme.php»

ü  Filename: «scheme_name.xsde» (имя файла схемы из каталога public/resource/schemes. Имя файла в конце содержит суффикс «.uni», в поле необходимо писать имя без этого суффикса).

 

ü  остальные поля (URL getConfig, URL getData, URL send, ID) оставить пустыми (незаполненными)

Сохранение схемы

Для сохранения схемы достаточно ввести в диалоге сохранения название новой схемы и нажать кнопку «ОК».

Эта схема сохранится c новым идентификатором.


Копирование источников данных

Для новой схемы не настроены источники данных. Чтобы избежать ручной настройки, можно воспользоваться ручным вызовом http-запроса к серверу iSMS.

Узнать новый идентификатор можно в параметрах схемы, как показано на рисунке ниже:

Чтобы скопировать источники данных, выполним http-запрос к серверу iSMS:

http://<адрес_isms>/json/newschemes/data-sources?action=clone&old_id=<id_схемы_modus>&id=<id_схемы_iSMS>

Замените <адрес_iSMS> на реальный адрес, <id_схемы_Modus> на идентификатор исходной схемы, <id_схемы_RTree> на идентификатор схемы.

Приложение

Запрос для получения полной информации по схемам и пунктам меню:

SELECT

    vs_form.id scheme_id,

    vs_form.name scheme_name,

    vs_form.file_name scheme_filename,

    vs_form.id_type scheme_type_id,

    sys_otyp.name scheme_type_name,

    menu.menu_id menu_id,

    menu.menu_name menu_name,

    menu.menu_user_id menu_user_id,

    menu.menu_user_login menu_user_login,

    menu.appl_id appl_id,

    menu.appl_name appl_name,

    menu.appl_alias appl_alias

FROM

    vs_form

    INNER JOIN sys_otyp ON

        sys_otyp.id = vs_form.id_type

    LEFT JOIN (

        SELECT

            us_menu.id_param scheme_id,

            us_menu.id menu_id,

            us_menu.name menu_name,

            us_menu.id_user menu_user_id,

            s_users.login menu_user_login,

            us_menu.id_appl appl_id,

            sys_appl.name appl_name,

            sys_appl.alias appl_alias

        FROM

            us_menu

            INNER JOIN sys_appl ON

                sys_appl.id = us_menu.id_appl

                AND sys_appl.alias IN ('sldiagram', 'schemeviewer.exe')

            LEFT JOIN sys_otyp ON

                sys_otyp.id = sys_appl.id_type

                AND sys_otyp.define_alias LIKE 'KLIENT_PO'

            INNER JOIN s_users ON

                s_users.id = us_menu.id_user

                /* AND s_users.login like 'admin' */

    ) menu ON

        menu.scheme_id = vs_form.id

Запрос для получения полной информации по элементам схемы и привязанным источникам данных:

SELECT

    vs_form.name scheme_name,

    vs_form.id scheme_id,

    d.comp_id,

    d.element_id_in_scheme element_id_in_scheme,

    d.element_id_in_database element_id_in_database,

    d.datasource_type datasource_type,

    d.sys_tbllst_name sys_tbllst_name,

    d.sys_gtopt_name sys_gtopt_name,

    d.vs_tcomp_name vs_tcomp_name

FROM

    vs_form

    LEFT JOIN (

        SELECT

            vs_comp.id_form scheme_id,

            vs_comp.id comp_id,

            vs_comp.name element_id_in_scheme,

            datasource.element_id_in_database element_id_in_database,

            datasource.datasource_type datasource_type,

            datasource.sys_tbllst_name sys_tbllst_name,

            datasource.sys_gtopt_name sys_gtopt_name,

            vs_tcomp.name vs_tcomp_name

        FROM

            vs_comp

            LEFT JOIN (

                SELECT

                    vs_regim_tune.id_ownlst comp_id,

                    vs_regim_tune.id_param element_id_in_database,

                    vs_regim.alias datasource_type,

                    sys_tbllst.name sys_tbllst_name,

                    sys_gtopt.name sys_gtopt_name

                FROM

                    vs_regim_tune

                    INNER JOIN vs_regim ON

                        vs_regim.id = vs_regim_tune.id_regim

                    LEFT JOIN sys_tbllst ON

                        sys_tbllst.id = vs_regim_tune.id_table

                    LEFT JOIN sys_gtopt ON

                        sys_gtopt.id = vs_regim_tune.id_gtopt

            ) datasource ON

                datasource.comp_id = vs_comp.id

            LEFT JOIN vs_tcomp ON

                vs_tcomp.id = vs_comp.id_tctrl

    ) d ON

        d.scheme_id = vs_form.id

WHERE

    vs_form.name LIKE '' /* <= заполнить название схемы */
