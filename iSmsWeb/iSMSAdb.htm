2.4         Подготовка приложений windows для переноса БД.

ü  Установить ODBC драйвер для MySQL.

ü  Создать ODBC источник в Windows для MySQL (Администрирование -> Источники данных (ODBC))

ü  Выбрать закладку «Пользовательский DSN» и нажать кнопку «Добавить…»

ü  В окне «Создание нового источника данных» выбрать драйвер MySQL ODBC.

Если драйвер MySQL ODBC отсутствует в списке, то следующие шаги вы выполнить не сможете. Попробуйте начать описание ODBC-источника с самого начала.

ü  В окне настройки соединения с источником данных указать: имя источника, IP адрес, порт оставить 3306, имя пользователя (root), пароль пользователя root, имя базы данных (rsduadmin).

2.5         Перенос БД

ü  Распакуйте архив FullConvert.zip.

ü  Запустите приложение FullConvert.exe, в котором следует проект ORA_TO_DB.fcs.

Примечание: При открытии проекта может возникнуть предупреждение, что невозможно соединение с БД Oracle или MySQL. В случае этих ошибок автоматически откроется окно редактирования настроек

ü  Выберите опцию "Edit project", как на показано на рисунке.

ü  На первых двух шагах открывшегося визарда настроить БД источник и БД приемник, остальные шаги пропустить нажатием "Next".

ü  После того как настроены параметры БД источника и приемника, необходимо настроить опции переноса. Для этого выбрать пункт "Home->Options". Откроется окно опций.

ü  В окне опций выберите закладку  "Transform". В полях "Table name case" и "Column name case" необходимо выбрать вариант "Lowercase".

ü  Перейдите к закладке "MySQL". В поле "Table engine" необходимо указать  "InnoDB", в поле "Character set" выбрать "utf8".

ü  Нажмите кнопку "Convert" и дождитесь завершения переноса.

ü  Убедитесь, что при переносе не произошло критических ошибок (иногда обнаруживается нарушение нескольких констрейнов, что не является критичным).

ü  Завершите приложение FullConvert.

ОПИСАНИЕ

 

Данная утилита предназначена для внесения изменений в базу данных MySQL после того, как сработает программа FullConvert. Цель работы - воссоздать структуру БД, аналогичную той, что имеется в БД Oracle, с которой снимается копия. Воссоздаются только те объекты, которые необходимы для запуска определенного списка приложений.

 

ПРЕДВАРИТЕЛЬНАЯ ПОДГОТОВКА

 

1. Заполнение файла "config.ini"

В первой строке файла содержится имя DSN, созданного на предыдущем шаге

В каждой следующей за первой строке содержится пара значений целого неотрицательного типа, конфигурирующая создание видов DA_V_CAT_ и DA_V_LST_ следующим образом:

первое число - это номер вида DA_V_CAT_ и DA_V_LST_,

второе число - это корневой элемент в таблице DA_CAT, от которого строится вид DA_V_CAT_

Пример: если нам необходимо, чтобы вид DA_V_CAT_3 строился от корневого узла с номером 10, то в файле "config.ini"

необходимо добавить строку

3 10

автоматически создастся вид DA_V_LST_3

Приложение 1. Дополнительные сведения

Дополнительные знания, которые могут понадобиться для решения возникших проблем.

1.         При необходимости (на этапе настройки) предоставить возможность для подключения к БД извне можно, изменив в файле /etc/mysql/my.cnf настройку адресов, прослушиваемых mysql:

# Instead of skip-networking the default is now to listen only on

# localhost which is more compatible and is not less secure.

#прослушивать только loopback

#bind-address           = 127.0.0.1

#прослушивать все адреса

bind-address            = 0.0.0.0

После настройки из соображений безопасности крайне рекомендуется вернуть настройку прослушки только на loopback:

# Instead of skip-networking the default is now to listen only on

# localhost which is more compatible and is not less secure.

bind-address           = 127.0.0.1

2.         Предоставление пользователю root прав подключаться с удалённого хоста с помощью команды:

mysql> grant all privileges on *.* to 'root'@'%' identified by 'passme' with grant option;

3.         Создание пользователя ‘dcsd_ga1’со всеми правами на БД ‘rsduadmin’, которому можно подключаться с локального хоста с паролем ‘rsdu_211’, можно выполнить с помощью команды:

mysql> GRANT EXECUTE, SELECT, SHOW VIEW, ALTER, ALTER ROUTINE, CREATE, CREATE ROUTINE, CREATE TEMPORARY TABLES, CREATE VIEW, DELETE, DROP, EVENT, INDEX, INSERT, REFERENCES, TRIGGER, UPDATE, LOCK TABLES  ON `rsduadmin`.* TO 'dcsd_ga1'@'localhost' IDENTIFIED BY 'rsdu_211';

4.         Проверить, что пользователь может зайти с локального хоста, можно с помощью команды:

$ mysql -hlocalhost -udcsd_ga1 -prsdu_211 rsduadmin

5.         Следующий скрипт генерирует SQL скрипт, который добавляет всех пользователей для процессов, описанных в БД.

select "GRANT EXECUTE, SELECT, SHOW VIEW, ALTER, ALTER ROUTINE, CREATE, CREATE ROUTINE, CREATE TEMPORARY TABLES, CREATE VIEW, DELETE, DROP, EVENT, INDEX, INSERT, REFERENCES, TRIGGER, UPDATE, LOCK TABLES "

union select "ON `rsduadmin`.*  TO"

union select CONCAT("'", login, "'@'localhost' IDENTIFIED BY 'rsdu_211',") from s_users as u inner join sys_otyp as ot on u.id_type=ot.id where u.login is not null and ot.define_alias='OTYP_PROCESS';

6.         Список серверов описанных на узле (1043) можно получить с помощью запроса

mysql> select login from s_users where id_node=1043 and login is not null;

 

 

                                               


Приложение 2. Сбрасывание паролей для процессов iSMS

Для того чтобы сбросить пароли, используемые серверными процессами iSMS, необходимо сделать следующее:

  1. Остановить серверные модули на сервере iSMS с помощью команды `ema stopww`
  2. Почистить пароли в конфигурационном файле /etc/ema/host.ini.

Для этого надо отредактировать файл /etc/ema/host.ini и удалить все строчки, начинающиеся с “PROCESS_LOGIN”, “PROCESS_PASSWORD”, “PROCESS_KEYTIME”. В этом же файле необходимо отредактировать строку “PROCESS_NUM” и указать значение 0 (ноль).

  1. Почистить пароли у пользователей в БД iSMS.

Для этого необходимо воспользоваться следующей процедурой.

DROP PROCEDURE IF EXISTS rsduadmin.clean_proc_passwords;

CREATE procedure rsduadmin.clean_proc_passwords()

begin

 DECLARE done INTEGER DEFAULT 0;

 DECLARE num INTEGER DEFAULT 0;

 DECLARE sLogin VARCHAR(255);

 

 DECLARE rCursor CURSOR FOR select login from s_users where (id_node=1043 OR id_node=1044) and login is not null order by id_node, login asc;

 DECLARE CONTINUE HANDLER FOR NOT FOUND SET done=1;

 

 SET num = 0;

 OPEN rCursor;

 FETCH rCursor INTO sLogin;

 WHILE done = 0 DO

  SET @CleanPasswordCommand = CONCAT ("grant all privileges on rsduadmin.* to '",sLogin,"'@'localhost' identified by 'rsdu_211'",";");

  SET num = num + 1;

  PREPARE stmt FROM @CleanPasswordCommand;

  EXECUTE stmt;

  FETCH rCursor INTO sLogin;

 END WHILE;

 CLOSE rCursor;

 SELECT num as "it was cleaned, passwords" FROM dual;

end;

call rsduadmin.clean_proc_passwords();

DROP PROCEDURE IF EXISTS rsduadmin.clean_proc_passwords;

  1. В завершение, нужно запустить серверные модули на сервере iSMS с помощью команды `ema start` и проконтролировать их запуск.
