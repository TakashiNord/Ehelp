Общие рекомендации по восстановлению БД MySQL

 

  1. Как можно быстрее прекратить запись в БД (и на раздел с БД).
  2. Сделать полную копию всех файлов БД (обычно из /var/lib/mysql) (лучше две копии, одна копия – та, над которой будут вестись работы по восстановлению, другая – резервная копия, к которой можно вернуться в случае неудачной попытки восстановления).
  3. После создания копий по возможности отключить раздел с повреждённой БД (или минимизировать запись на неё).
  4. Все попытки восстановления сперва предпринимать с копией.
  5. В случае успешного запуска MySQL с восстановленной копией БД снять полный дамп БД утилитой mysqldump.
  6. Предпринять попытки по восстановлению оригинала БД тем же способом, что был проделан ранее, или выполнить импорт БД из дампа, полученного в п. 5.
