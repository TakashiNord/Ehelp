1. Выбираем панели, которые необходимо отображать в WEB и помещаем их в предпочтения пользователя.

PS. Имя файла панели должно иметь только латинские буквы!

    Иначе необходимо выполнить переименование имени файла панели на сервере ресурсов и исправить это название в таблице VP_PANEL.

 

2. Копируем uir-файлы в каталог с утилитой преобразования панелей.

 

3. Генерируем WEB-панели с помощью приложения CviPanelParser.exe (запуск через bat-файл AllUir2Js.bat)

На выходе должны быть файлы с расширением *.js

 

4. Копируем файлы в каталог на сервере WEB-iSMS в каталог /home/www-data/www/htdocs/WASUTP/public/resource/panels/

 

5. Даем файлам права на чтение с помощью команды

chmod +r *.js

 

6. Если необходимо сгенерировать отображение "сложной" схемы с подложкой, то необходимо отдельно сгенерировать

"голую" панель с отображением ТИ и подолжку в виде картинки.

6.1 Чтобы сгенерировать панель только с ТИ, то нужно в файле AllUir2Js.bat использовать вариант запуска утилиты с опцией n

6.2 Фотографируем панель и сохраняем картику в формате *.png

 

6.3 Прописываем в файле *.js путь до картинки (см. пример)

Ext.define('MyPanels.ATEC_TAI_KOMDIS', {

id: 'ATEC_TAI_KOMDIS',

title: 'Р­РєСЂР°РЅ',

width: 1216,

height: 672,

layout: { type: 'absolute'},

bodyStyle: 'background:#000000 url("/ISMSBACK.png") left top no-repeat;',

 

6.4 Помещаем файл подложки в директорию /home/www-data/www/htdocs/WASUTP/public/

 

7. Заходим в WEB-портал от имени того пользователи, чьи панели необходимо просматривать.

PS. Если были обновлены файлы WEB-панелей, то необходимо обновить страницу, сбросив закешированные в WEB-браузере картинки, нажав Ctrl+F5.
