iSMS 1.19.1.2 (06.12.2021)
==========================
Новые функции:

* Подсистема сбора. Протоколы МЭК101/103/104: Добавлена новая опция "Часовой пояс для синхронизации времени (SYNC_TIME_ZONE_DIFF)", значение задаётся в секундах. Если опция SYNC_TIME_ZONE_DIFF задана, то для синхронизации времени контролируемому измерителю передаётся метка времени в указанном часовом поясе, то есть UTC + SYNC_TIME_ZONE_DIFF.
* Подсистема сбора. Протокол PML ION: Добавлены новые опции: "Ожидание (T1) устройства на канальном уровне (LINK_TIMEOUT)" в секундах и "Повторов кадра при ошибке в канале (LINK_RETRYES)".
* Отчёты. Реализовано общее приложение "Архивы" для просмотра архивов по параметрам, которое запускается через пункт главного меню "Отчеты->Архивы->Архивы ...".
* Документация. Добавлен раздел с описанием приложения для просмотра архивных данных.
* Документация. Добавлено описание новых опций для протоколов.

Улучшения:
* Архивы. Увеличена максимальная глубина хранения архивов до 2-х лет.
* Интерфейс. Приведено к единому стилю обозначение секунд в названиях опций.
* Интерфейс. Заменена иконка в контекстном меню "Показать виджет" в окне трассировки.

Исправлены ошибки:
* Инсталлятор iSMS. Исправлена ошибка при обновлении пакета isms-db через команду 'dpkg -i'.

iSMS 1.19.1.0 (25.03.2019)
==========================
Новые функции:
* Подсистема сбора. Ключ Телеуправления: реализация распределения функций и прав Телеуправления между АРМ ПС, ОИК ДЦ и ПТК ЦУС в соответствии с "Типовыми техническими требованиями к ПТК АСУ ТП подстанций и к обмену технологической информацией для осуществления функций телеуправления оборудованием и устройствами РЗА подстанций из диспетчерских центров АО "СО ЕЭС" и центров управления сетями сетевых организаций".
* Подсистема сбора. Протокол Modbus TCP в режиме сервера: Добавлена новая опция "Ожидание устройства на канальном уровне (LINK_TIMEOUT)", значение задаётся в секундах. Опция LINK_TIMEOUT задаёт интервал запуска проверки активности соединения (TCP Keep-alive). Если в течение следующих 30 секунд канал бездействует, соединение закрывается. По умолчанию опция LINK_TIMEOUT равна 60 секундам.
* Подсистема сбора. Реализован быстрый переход к параметрам подсистемы сбора из профиля сбора с помощью соответствующей команды контекстного меню.
* Подсистема сбора. Реализована функция поиска параметра по ID.
* Редактор схем. Добавлен новый элемент "Подстанция" в палитру графических элементов с возможностью настройки отображения нескольких уровней напряжения; поддерживается импорт элемента из схем формата МОДУС.
* Редактор схем. Добавлен новый графический элемент "Направление перетока" в виде направленной стрелки; поддерживается импорт элемента из схем формата МОДУС. "Направление перетока" настраивается на аналоговый параметр.
* Редактор схем. Добавлен новый элемент "Вольтодобавочный трансформатор" в палитру графических элементов; поддерживается импорт элемента из схем формата МОДУС.
* Редактор схем. Добавлен новый элемент "Круг" в палитру графических элементов.
* Редактор схем. Добавлена возможность выбора дополнительных цветов "Класс напряжения" и "Прозрачный" (цвет фона) в качестве цвета состояний ВКЛЮЧЕНО/ОТКЛЮЧЕНО для выключателей. Поддерживается импорт данных цветов из схем формата МОДУС.
* Интерфейс. Реализована функция трассировки параметров подсистемы сбора. Пункт меню "Трассировка" позволяет оперативно посмотреть, в каких подсистемах используется данный параметр (схемы, архивы, каналы, источники и т.п.), и перейти в нужную точку.

Улучшения:
* Подсистема сбора. Протокол Modbus TCP в режиме сервера: Добавлена сигнализация о неисправности передачи и фиксации события в журнале неисправностей измерителей подсистемы сбора и передачи.
* Подсистема сбора. При создании нескольких измерителей на одном порту проверяется уникальность введённого физического адреса.
* Подсистема сбора. Оптимизирована операция удаления прибора с большим количеством параметров (больше 1000).
* Подсистема сбора. Протокол МЭК-61850 MMS: Добавлено поле "Путь" для отчёта фиксированной конфигурации профиля 61850.
* Подсистема сбора. Протоколы МЭК101/103/104: Добавлена отправка подтверждения исполнения ТУ при ретрансляции команд ТУ.
* Интерфейс. В окно "Информация о системе" добавлена версия браузера клиента.
* Интерфейс. В дереве сбора и передачи выделяется красным шрифтом порт/устройство сбора, если не задан либо номер порта, либо IP-адрес при неустановленном флаге "Сервер (ждем соединения)".
* Интерфейс. Запрещено телеуправление для учётной записи Администратор (administrator). Теперь телеуправление разрешено только Диспетчеру (dispatcher) и Администратору БД (supervisor).
* Редактор схем. Добавлена автоматическая подстановка наименования подсистемы в источниках данных при перетягивании параметра из подсистемы сбора.
* Редактор схем. Для элемента "Прямоугольник" добавлена возможность редактирования и сохранения свойства "Стиль линии" с вариантами "Сплошная" или "Пунктирная", добавлена возможность выбора "прозрачного" цвета фона и линии.
* Редактор схем. Добавлено отображение имени параметра в настройках источников с возможностью перехода к параметру.
* Инсталлятор iSMS. Обновлена СУБД MySQL до версии 5.7 для поддержки нового функционала iSMS.

Исправлены ошибки:
* Подсистема сбора. Протокол МЭК103: Доработан алгоритм синхронизации времени на измерителях: теперь отрицательный результат синхронизации времени не влияет на дальнейший опрос измерителя.
* Подсистема сбора. Протоколы МЭК101/104: исправлена ошибка ретрансляции команды ТУ в случае приёма команды ТУ на резервном канале (активный резерв).
* Подсистема сбора. Протоколы МЭК101/104: Убрано лишнее присваивание признака недостоверности (бит NT) при ретрансляции TC двухэлементной информацией при неопределённом или промежуточном состояниях.
* Подсистема сбора. Доработано отображение свойств параметров: исправлены заголовки окон при открытии свойств источников, исправлен интерфейс по установке свойства "Инверсия" для ТС, а также доработан интерфейс по заданию номера бита при обработке ТС.
* Подсистема сбора. Добавлена очистка файла дорасчёта при импорте новой конфигурации (без файла дорасчёта) поверх старой (с файлом дорасчёта). Отсутствие очистки в подобном случае приводило к ошибкам сбора (так как использовался файл дорасчёта от старой конфигурации).
* Просмотр/редактор схем. Исправлена ошибка изменения масштаба (комбобокс "Выбор масштаба").
* Редактор схем. Исправлена ошибка отображения рамки выделения (при работе со схемой в какой-то момент переставала появляться рамка выделения при выборе элемента).
* Редактор схем. Исправлено позиционирование текста при импорте из схем формата МОДУС.

iSMS 1.18.1.0 (28.09.2018)
==========================
Новые функции:
* Подсистема сбора. Стандарт МЭК-61850: Реализован сбор по протоколу "МЭК-61850 MMS"; для описания сигналов в протоколе "МЭК-61850 MMS" предназначен новый сегмент сбора "Сбор IEC61850".
  Описание функций, которые не поддерживаются в данной версии, приведено в Руководстве оператора по программному обеспечению ПТК телемеханики iSMS.
  Ограничения в работе МЭК-61850:
  - при получении буферизированных отчетов в режиме резервирования после остановки модуля dsgwd на основной сервер приходит весь буфер значений для параметра, а на резервный - только последнее значение;
  - при получении буферизированных отчетов после потери сетевого соединения с устройством после включения возможно получение не только данных, которые изменились во время отключения, но и тех, которые уже были высланы ранее;
  - если в сегменте сбора и передачи "Сбор IEC61850" введено в работу устройство, у которого не задан IP-адрес, то после отправки  с него команды телеуправления выгружается серверный модуль dsgwd.
* Подсистема сбора. Протоколы МЭК101/104: Добавлены опции "Буферизация ТС (BUFFER_LENGTH_TS)" и "Буферизация ТИ (BUFFER_LENGTH_TI)" для направлений на передачу; опции задают максимальное количество изменений каждого ТС и ТИ, которые могут быть накоплены для передачи в случае кратковременного отсутствия связи. Значения по умолчанию: 1024 (ТС), 0 (ТИ).
* Подсистема сбора. Протоколы МЭК101/103/104: Добавлена новая опция "Общий адрес АСДУ (COMMON_ADDR_ASDU)": в случае если она задана, то переопределяется общий адрес АСДУ, указанный в профиле измерителя.
* Подсистема сбора. Протоколы МЭК101/103/104: Добавлена новая опция "Таймаут на установку соединения (CONNECTION_TIMEOUT)". Опция задаёт таймаут на подключение, задаётся в секундах. Значение по умолчанию: 30 (сек.).
* Подсистема сбора. Протокол МЭК101: Добавлена новая опция "Соотношение высокоприоритетных и низкоприоритетных значений (N:1) (SEND_HPSP_RATIO)" для направлений МЭК101, настроенных на передачу. Значение по умолчанию: 4.
* Подсистема сбора. Протоколы МЭК101/104: Определён новый сигнал "Сформирована осциллограмма" (SS_RCVD_OSCILLOGRAM): сигнал посылается в случае приёма по протоколу МЭК101/104 от измерителя кадра с типом F_DR_TA_1 (126).
* Подсистема сбора. Протоколы МЭК101/104: Добавлена настройка функции резервирования портов (каналов связи): в свойствах описания узла с типом "Порт" появилась возможность выбора основного порта.
* Подсистема сбора. В свойствах профильной точки добавлен параметр "Коэффициент масштабирования", который копируется в свойства создаваемых параметров подсистемы сбора и передачи.
* Подсистема сбора. В свойствах параметров ТУ, созданных на устройствах сбора, добавлена вкладка "Источники блокировки" для задания источников блокировки телеуправления.
* Подсистема сбора. Для двухпозиционных ТС введены новые состояния: "Промежуточное (нейтральное) положение" и "Неисправность (положение не определено)".
* Интерфейс. Добавлено отображение новых состояний для ТС: "Промежуточное положение" и "Неисправность" на виджетах (отображается как ПРОМЕЖУТ, НЕИСПР) и во всплывающих подсказках к параметру в окне подсистемы сбора.
* Отчёты. Разработан стандартный журнал "Журнал телеуправления", который содержит события по обработке (приёму, ретрансляции) команд телеуправления.
* Инсталлятор iSMS. Реализовано автоматическое применение обновлений БД iSMS при обновлении пакета isms-db.
* Инсталлятор iSMS. В дистрибутив iSMS добавлены новые пакеты bridge-utils и quagga для расширенной конфигурации сети.
* Документация. В Руководство по эксплуатации ПТК телемеханики iSMS добавлено описание настройки сетевого моста (bridge) из нескольких сетевых интерфейсов.

Улучшения:
* Редактор схем. Добавлен импорт из схемы формата МОДУС размера шрифта цифрового табло.
* Редактор схем. Добавлена обработка начального состояния для элементов "Разъединитель-предохранитель" и "Лампа сигнальная" при импорте из схемы МОДУС.
* Редактор схем. Добавлена обработка ориентации для элементов "Конденсатор" и "Разъединитель напряжения" при импорте из схемы МОДУС.
* Интерфейс. Добавлена визуализация выполнения операции экспорта конфигурации iSMS.
* Интерфейс. Добавлена кнопка "Очистить консоль" в окнах "Консоль ошибок подсистемы сбора" и "Консоль ошибок модуля дорасчёта".
* Интерфейс. Добавлена возможность редактирования имени виджета, отображаемого на рабочем столе iSMS (пункт контекстного меню "Переименовать" для компонента виджета).
* Интерфейс. Добавлена проверка состояния комплекса при попытке смены статуса сервера через WEB-клиент: если статус сервера "неопределенный", функция смены статуса недоступна.
* Интерфейс. В уведомлениях системы сообщение "Изменения приняты" теперь содержит информацию о подсистеме, к которой были применены изменения.
* Инсталлятор iSMS. Добавлена проверка на корректность введённого IP-адреса узла iSMS.

Исправлены ошибки:
* Подсистема сбора. Исправлена репликация данных между основным и резервным сервером iSMS: после смены режимов основной/резервный на резервном сервере могли оставаться недостоверные ТелеСигналы до тех пор, пока состояние не изменится снова на основной.
* Подсистема сбора. Исправлена ошибочная установка статуса "Нарушен обмен" у параметров при работе iSMS в резервном режиме.
* Подсистема сбора. Исключена возможность описания одинаковых опций у измерителя (например, в диалоге добавления опции можно было несколько раз подряд нажать кнопку ОК и на каждое нажатие выполнилось бы добавление); аналогичная ошибка исправлена в диалогах создания параметров сбора и передачи.
* Редактор схем. Добавлено свойство "Номер слоя" для "Реактор шунтирующий", "Заградительный фильтр", "ОПН без земли" (ранее отсутствовало).
* Редактор схем. Исправлен конвертер схем формата МОДУС: для элемента "Шина" добавлен перенос свойства "Толщина".
* Редактор схем. Исправлен конвертер схем формата МОДУС: для элемента "Пересечение провода" добавлен перенос свойства "Размер"; также скорректировано изображение дуги для элемента "Пересечение провода".
* Редактор схем. Исправлен импорт размера элемента "Выключатель" при импорте из схем МОДУС.
* Редактор схем. Исправлен импорт неподдерживаемого элемента "Направление перетока" (при импорте из схем МОДУС он преобразовывался в "Контакт дугогасительный", что не соответствует действительности).
* Просмотр схем. Исправлен вызов контекстного меню для графических элементов, которые накладываются друг на друга.
* Инсталлятор iSMS. Исправлена ошибка при обновлении пакета isms-server через команду 'dpkg -i'; добавлена возможность изменения выбранного на этапе установки номера сервера с помощью команды 'dpkg-reconfigure isms-server'.

iSMS 1.16.2.0 (29.08.2016)
==========================
Новые функции:
* Редактор схем. Для графических элементов в настройках источников добавлены свойства "Коэффициент" и "Инверсия", которые влияют на отображаемое значение в элементе схемы. * Инсталлятор iSMS. Поддержка нового часового пояса для Новосибирской области (изменения вступили в силу с 24.07.2016).

Улучшения:
* Просмотр схем. Все настроенные на отображение значений элементы при открытии схемы объявляются недостоверными, до тех пор, пока не придут реальные значения.

Реализовано сохранение порядка, видимости столбцов, сортировки и ширины столбцов в таблицах приложений "Подсистема сбора", "Профили", "Архивы" и "Системные сообщения". Теперь пользователь может перегрузить страницу браузера и настройки отображения таблиц останутся прежними.
* Подсистема сбора. Добавлен запрос IP-адреса и IP-порта при создании измерителя сбора с протоколом МЭК104.
* Редактор схем. Доработано отображение значений в булевых и аналоговых элементах схемы. При настройке булевого источника на цифровой прибор, будет отражаться либо 0 либо 1. При настройке аналогового источника на булевый элемент, будет отображаться положение "ОТКЛ" (в случае значения <=0), либо "ВКЛ" (в случае значение >0).
* Редактор схем. Добавлено отображение цвета для свойства элементов "Класс напряжение".
* Редактор схем. Обновлён список курсоров, используемых при работе со схемой.
* Редактор схем. Для элемента "Цифровой прибор" изменена обработка свойства "Значение": введённое значение теперь отображается без учёта ограничения в "Число знаков". Также в качестве разделителя дробной части используется точка вместо запятой.
* Просмотр панелей. Добавлена обработка новых элементов: "циферблат" и "переключатель" в конвертере CVI-панелей.
* Изменены права для запуска приложения "SQL-консоль". Теперь "SQL-консоль" может быть запущена не только суперпользователем, но и любым другим пользователем, обладающим привилегией настройки ресурса "SQL-консоль".

Исправлены ошибки:
* Подсистема сбора. Исправлен сброс счётчика квалификатора после удаления точек из профиля передачи.
* Редактор схем. Исправлено "притормаживание" во время перемещения элементов схемы с помощью клавиш клавиатуры ("вверх", "вниз", "влево", "вправо").
* Редактор схем. Исправлена обработка размеров многострочного текста: вычисляется реальный размер ширины текстового элемента, таким образом, не происходит перекрытие соседних элементов.

Улучшения:
* Добавлена возможность опроса нескольких устройств MODBUS на одной магистрали.
* Добавлена возможность отключения направления выводов у силовых трансформаторов.
* Обновлены иконки для ярлыков программ, помещаемых на рабочий стол iSMS.
* Добавлено удаление ярлыков с рабочего стола при удалении исходных схем/карт в iSMS.
* При начальной установке iSMS устанавливается стандартный видеорежим 640x480 для использования с любыми мониторами.
* Настроенные элементы на схеме отображаются со статусом недостоверности до первого цикла опроса.
* Изменения в интерфейсе создания профиля общих функций (контроль каналов, двухпозиционные ТС, преобразование типов).
* Изменено отображение статусных рамок и статусных символов в просмотрщике схем.

Исправлены ошибки:
* Исправлено описание опций у измерителя - одну и ту же опцию можно описать только один раз.
* Исправлена установка физических адресов по умолчанию при создании измерителей MODBUS/МЭК101/МЭК104.
* Исправлено перемещение по схеме в режиме просмотра: корректная работа перемещения при попадании мышью по элементу на схеме.
* Исправлено удаление виджета с рабочего стола при удалении параметра из подсистемы сбора.
* Исправлен внешний вид элемента "Шасси".
* Исправлено обновление списка параметров в подсистеме сбора и передачи при перемещении по узлам дерева по щелчку как левой, так и правой кнопок мыши.
* В отчетах iSMS исправлены граничные условия на фильтр по дате, чтобы исключить в результате выборки записи из будущих суток (несоответствие наблюдалось на границе суток).
* Исправлена ошибка невозможности создания двух ярлыков с одинаковым именем.
* Исправлена ошибка разбиения диска при начальной установке iSMS.
* Исправлен порядок загрузки/выгрузки сервисов iSMS при загрузке/остановке операционной системы.
* Исправление отображение недостоверного статуса у элемента схемы при некорректной настройке.
* Исправлено копирование настроенных на источники элементов схемы.
