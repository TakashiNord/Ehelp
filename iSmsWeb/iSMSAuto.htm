Как устанавливать iSMS автоматически

 

ОС Debian умеет устанавливаться автоматически используя т.н. preseed-файл.

Это файл в котором хранятся значения переменных,

которые в обычном случае будут спрошены у программиста во время установки.

 

Так как установка iSMS обычно типичная, большинство параметров не меняются.

 

Этот файл можно "вшить" в iso образ, что потребует его переупаковки, но тогда

установщик не задаст никаких вопросов.

 

Второй вариант - передать ядру параметр с указанием url откуда взять preseed файл

для установки. Ядро может получить этот файл по http. Именно этот вариант и будет

рассмотрен.

 

В целом настройка выглядит так:

  1. Готовим preseed.conf
  2. Устанавливаем iSMS, чтобы она была в локальной сети к которой есть доступ
  3. Проверяем iSMS по сети, если нужно корректируем настройки по ssh, чистим следы
  4. Заливаем настройки сети по умолчанию
  5. Непосредственно на iSMS, подключив монитор, проверяем, что все сработало

Как подготовить preseed.conf

 

За основу необходимо взять файл из дистрибутива, который скорее всего лежит в:

isms.iso/distrib/install.amd/preseed.conf

isms.iso/distrib/install.386/preseed.conf

 

Добавить в этот файл строки из файла, что лежит в папке Automated installation resources/auto.conf:

 

cp path/to/original/preseed.conf preseed.conf

cat Automated\ installation\ resources/auto.conf >> preseed.conf

 

Теперь файл preseed.conf готов к работе.

Установка и проверка iSMS

1.      Выбрать необходимый iso образ iSMS и записать его на флешку или cd

 

2.  На локальном компьютере запустить http-сервер и положить в его корень подготовленный preseed.conf файл. Простейший способ это запустить папке с файлом:
sudo python3 -m http.server --bind <ip адрес компа в сети> 80

 

3.      В самом начале работы у загрузчика выбрать нужный вариант установки и нажать tab для редактирования параметров ядра и заменить параметр file=/.../preseed.conf на 'auto url=http://<ipaddress>/preseed.conf'

 

4.      Для продолжения установки необходимо выбрать язык и настроить сеть:.

4.1.   Выбрать русский язык.

4.2.   Для настроек сети проще всего выбрать автоматическую настройку, также известную как dhcp. Главное, чтобы iSMS и http-сервер оказались в одной сети, так удобнее тестировать

5.      После выбора настроек сети и языка начнется установка и по сети будет получен preseed.conf, что можно будет увидеть в логах веб сервера. Дальше установка производится автоматически и не требует ручного вмешательства.

6.      По окончании установки проверить через браузер работоспособность сервера iSMS (можно, например, использовать ноутбук с подключением через перекрестный кабель, а можно и по локальной сети):

6.1.   Описать один измеритель для сбора информации;

6.2.   Настроить запись архивов для ТС и ТИ;

6.3.   Вывести на схему отображение одного параметра ТС и одного параметра ТИ;

6.4.   Вывести и сохранить виджеты на рабочем столе;

6.5.   Проверить запись и отображение настроенных архивов через отчеты iSMS;

6.6.   Проверить перезагрузку серверных модулей через веб-интерфейс iSMS;

6.7.   Проверить перезагрузку сервера iSMS через веб-интерфейс iSMS;

 

7.      На данной стадии, если нужно установить iSMS на несколько устройств можно сохранить конфигурацию iSMS через веб-интерфейс на компьютер и для следующих проверок использовать именно его. Перед загрузкой конфигурации в веб, нужно сделать бекап конфигурации по умолчанию, чтобы было потом из чего восстанавливать.

 

8.      Проверки связанные с железом

8.1.   iSMS должна включаться автоматически при подключении питания

8.2.   Lsmagentd должен перезагружать iSMS при отключении сети.

8.3.   Проверить, что не происходит остановок при многократных перезагрузках. Можно просто отключить iSMS от сети, и тогда lsmagentd сделает всю работу, останется только следить за тем что перезагрузки действительно происходят.

 

9.      По окончании проверок почистить следы:

9.1.        Восстановить конфигурацию в веб-интерфейсе, если она была изменена

9.2.        Удалить созданные для проверки ресурсы (схемы, измерители в подсистеме сбора, архивы, виджеты и пр):
rm /home/www-data/www/htdocs/WASUTP/public/resource/schemes/*.sch

9.3.        Почистить логи серверного ПО: ema stop all; ema dl

 

10.  Восстановить настройки сети по умолчанию, так как сейчас они или были получены по dhcp, или были заданы при установке. Можно просто заменить /etc/network/interfaces файлом с новыми настройками, скопировав по scp(ssh) на iSMS. Файл interfaces пригодный для использования(возможно стоит изменить подсеть) можно найти в папке Automated installation resources/interfaces
