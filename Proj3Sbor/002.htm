1 Общие сведения

1.1 Назначение оперативного расчета значений в сервере DCS

Оперативный расчет значений в сервере DCS предназначен для вычисления значений параметров распределенной системы сбора при помощи арифметических или логических выражений, вложенных подпрограмм и встроенных функций. Оперативный расчет значений в сервере DCS реализован на базе скриптового языка программирования LUA и позволяет использовать все его возможности. Функция оперативного расчета значений выполняется в каждом технологическом цикле сервера DCS. При каждом запуске происходит считывание и выполнение подготовленного скрипта, содержащего формулы и алгоритмы расчета. Результатом работы скрипта являются вычисленные значения параметров распределенной системы сбора. Все внесенные изменения в скрипт с формулами и алгоритмами принимаются «на лету» и становятся актуальными уже на следующем технологическом цикле сервера DCS.

В Приложение А приведен перечень функционала РСДУ, реализованного с помощью оперативного расчета значений в сервере DCS.

2 Настройка оперативного расчета в сервере DCS

Настройка оперативного расчета значений выполняется в три этапа:

1.    Конфигурирование профиля распределенной системы сбора для функций оперативного расчета в сервере DCS;

2.    Конфигурирование виртуального прибора оперативного расчета сервера DCS;

3.    Создание исполняемого скрипта оперативного расчета значений.

2.1 Конфигурирование профиля распределенной системы сбора для функций оперативного расчета в сервере DCS

Функция оперативного расчета значений является внутренней функцией сервера распределенной системы сбора DCS. Создание параметров, значения которых должны рассчитываться, начинается с описания профиля виртуального прибора «Дорасчет» в приложении «Справочная информация по распределенной системе сбора данных» (DAView.exe)  (см. документ «Описание параметров в подсистеме сбора и передачи РСДУ. Руководство администратора»).

Перед началом работы следует проверить наличие протокола «Доступ к функциям модулей» в справочнике протоколов. Проверка выполняется вызовом пункта «Протоколы» в меню «Справочники» в приложении «Справочная информация по распределенной системе сбора данных».

внимание!

Протокол «Доступ к функциям модулей» должен иметь в справочнике DA_PROTO_DESC идентификатор 72.

Из любого клиента Oracle необходимо выполнить запрос: select name from da_proto_desc where id=72

Если под этим идентификатором зарегистрирован другой протокол, необходимо сообщить разработчикам, чтобы актуализировали описание в справочнике подсистемы сбора.

2.2 Конфигурирование виртуального прибора оперативного расчета сервера DCS

После заполнения профиля распределенной системы сбора для функций оперативного расчета можно приступить к созданию параметров в сегменте сбора, который обслуживается соответствующим модулем коммуникационного сервера сбора и передачи DCS. Значения созданных параметров будут вычисляться в каждом технологическом цикле. Данные параметры, как и любые другие параметры распределенной системы сбора, могут быть использованы в качестве значений параметров электрического режима, коммутационных аппаратов, СРЗиА и других технологических серверов. При необходимости, рассчитываемые параметры можно сохранять в архивах распределенной системы сбора.

Проверьте наличие виртуального прибора в выбранном сегменте сбора, протоколом которого является «Расчет значений параметров». При отсутствии такого прибора или при необходимости разнести функции дорасчета по нескольким приборам, такой прибор надо создать. Никаких дополнительных настроек в описании порта для данного прибора и самого прибора не требуется (см. документ «Картридж "Распределенная подсистема сбора данных". Руководство администратора»).

Примечание. Количество приборов на основе протокола «Расчет значений параметров» не ограничено.

2.3 Создание исполняемого скрипта оперативного расчета значений сервера DCS

Исполняемый скрипт является текстовым файлом и может создаваться любым текстовым редактором. Имя текстового файла должно соответствовать имени загружаемого модуля DCS. Расширение файла – LUA. Например, если на хосте работают несколько модулей DCS  с именами dcs_main, dcs_test, dcs_ktp, то каждый из них должен иметь собственный файл с именами соответственно dcs_main.lua, dcs_test.lua, dcs_ktp.lua.

Исполняемые скрипты располагаются в папке /home/ema на сервере, на котором работают модули DCS.

 

3 Описание функций, используемых в оперативном расчете сервера DCS

Дорасчет LUA в DCS позволяет использовать все возможности скриптового языка LUA. Далее приведены основные наиболее часто востребованные функции и примеры их применения в РСДУ5.

3.1 Служебные таблицы

data[] – глобальная таблица для доступа к параметрам, ключом является идентификатор из подсистемы сбора и передачи. Читать данные или статусы можно из любого параметра подсистемы сбора и передачи, запись возможна только в параметры, которые не имеют источников данных.

global[] – глобальная таблица для нужд пользователя, её содержимое не сбрасывается по окончанию технологического цикла. В случае если по заданному ключу в таблице нет данных, возвращается 0.

3.2 Функции для объектов из таблицы data[]

getStatus() – возвращает текущий статус параметра.

getValue() – возвращает текущие данные параметра.

setStatus(number status) – сбрасывает все установленные ранее статусы параметра и устанавливает новый указанный статус.

setValue(number value) – задаёт новое значение данных для параметра. В случае попытки установить значение INF или NaN, значение принимает 0, и устанавливается статус «Данные недостоверны».

update() – отсылка новых статусов и данных параметра, эту функцию необходимо вызывать после окончания всех операции с помощью функций setStatus и setValue.

--Установить статус параметра с id 518 как побитовое ИЛИ статусов

--из параметров с id 73 и 74

data[518]:setStatus(data[73]:getStatus() | data[74]:getStatus())

 

--Вычисление данных для параметра с id 518 из данных параметров с id 73 и 74

data[518]:setValue((data[73]:getValue()^2 + data[74]:getValue()^2)^0.5/2)

 

--Передать выше сделанные изменения для параметра с id 518

data[518]:update()

 

Также существует более простой способ работы с данными, используя непосредственное обращение к объектам из таблицы data[]. Все операции выполняются со значениями, вычисление статуса происходит автоматически путём побитового ИЛИ между статусами операндов, а также происходит автоматическая отсылка обновления.

--Вычисление данных для параметра с id 519 из данных параметров с id 73 и 74, вычисление статуса и отсылка обновления происходят автоматически

data[519] = (data[73]^2 + data[74]^2)^0.5/2

 

--Данные параметра с id 520 принимает 42, все статусы сбрасываются, изменения

--автоматически переданы

data[520] = 42

setControl(boolean value) – задать состояние параметра телеуправления. Отсылка обновления будет произведена автоматически.

3.3 Операции для объектов из таблицы data[]

С объектами из таблицы data[] возможны следующие операции:

ü  Арифметические:

«+» – сложение

«-» – вычитание

«*» – умножение

«/» – деление

«^» – возведение в степень

ü  Логические:

«|» – логическое ИЛИ

«&» – логическое И

«~» – инверсия

3.4 Функции для работы со статусами для объектов из таблицы data[]

setValueNoCorrect() – установить статус «Данные недостоверны».

setNoPresent() – установить статус «Источник данных отсутствует».

setDeratingWarning() – установить статус «Нарушены предупредительные уставки».

setDeratingAlarm() – установить статус «Нарушены аварийные уставки».

setNoScanned() – установить статус «Выведен из работы».

setManual() – установить статус «Ручной ввод значения».

setSlaveSend() – установить статус «Ведется передача во внешние системы».

setSubstitution() – установить статус «Признак замещения параметра».

setSrcNoCorrect() – установить статус «Признак некорректной работы драйвера».

setDevNoCorrect() – установить статус «Признак некорректного источника данных».

После установки всех необходимых статусов нужно отослать итоговый обновлённый статус с помощью функции update.

if data[523]:getValue() > 225 then

    data[523]: setDeratingWarning()

    data[523]: setValueNoCorrect()

else

   data[523]: setStatus(0)

end

data[523]:update()

isValueNoCorrect() – проверка статуса «Данные недостоверны».

isNoPresent() – проверка статуса «Источник данных отсутствует».

isDeratingWarning() – проверка статуса «Нарушены предупредительные уставки».

isDeratingAlarm() – проверка статуса «Нарушены аварийные уставки».

isNoScanned() – проверка статуса «Выведен из работы».

isManual() – проверка статуса «Ручной ввод значения».

isSlaveSend() – проверка статуса «Ведется передача во внешние системы».

isSubstitution() – проверка статуса «Признак замещения параметра».

isSrcNoCorrect() – проверка статуса «Признак некорректной работы драйвера».

isDevNoCorrect() – проверка статуса «Признак некорректного источника данных».

3.5 Константы

Для описания условий с параметрами, имеющими тип телесигнализация, можно использовать предопределённые константы ON и OFF.

-- Проверка условия ТС включён

if data[88] == ON then

     data[520] = data[73]+data[74]

end

 

-- Проверка условия ТС выключен

if data[88] == OFF then

     data[520] = data[73]*data[74]

end

5 Администрирование конфигурационного файла оперативного расчета сервера DCS

Администратору комплекса РСДУ5 необходимо контролировать актуальность и идентичность исполняемых скриптов на основном и резервном полукомплекте РСДУ5. Для этого рекомендуется хранить текстовый файл скрипта на сервере ресурсов в папке R:\lua. Редактирование файла выполняется текстовым редактором Notepad++. Для его последующей рассылки на серверы с работающими DCS рекомендуется подготовить командный файл Lua.bat:

pscp -P 22 -scp -pw passme R:\lua\dcsd.lua root@10.5.165.20:/home\ema\dcsd.lua

pscp -P 22 -scp -pw passme R:\lua\dcsd.lua root@10.5.165.30:/home\ema\dcsd.lua

pause

где 10.5.165.20, 10.5.165.30 – IP-адреса основного и резервного хостов, на которых работает сервер dcsd, root – пользователь для подключения к хосту по SSH.

Командный файл lua.bat можно описать в программном обеспечении РСДУ5 и осуществлять его запуск из Панели оператора, настроив предпочтения (Рисунок 17, Рисунок 18). Также возможно настроить его запуск с кнопок с информационной панели. В таком случае командный файл lua.bat необходимо располагать в папке R:\BIN.

внимание!

При расположении файла lua.bat в каталоге R:\bin необходимо его сохранять перед установкой нового релиза РСДУ и выкладывать в каталог после окончания установки, т.к. данный каталог полностью перезаписывается при установке нового релиза.
