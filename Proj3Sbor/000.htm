3 Добавление параметра в профиль протокола IEC60870-5-101(104)

Создание и конфигурирование профиля устройства производится с помощью утилиты DAView (Справочная информация по распределенной системе сбора). Какие функции будет выполнять данное устройство (КП или ПУ) не имеет значения. Пример для протокола МЭК870-5-101, для МЭК870-5-104 действия аналогичны.

Рисунок 6 – Просмотр информации о протоколе

1.    Выбрать в меню «Справочники» пункт «Протоколы». В открывшемся окне «Протоколы» проверить, что информация о протоколе МЭК внесена в список (как IEC60870-5-101). Здесь определяется структура адресации информационного объекта (точки) в протоколе МЭК870-5-101.

2.    Выбрать в меню «Справочники» пункт «Устройства сбора». В открывшемся окне «Устройства сбора» создать устройство, которое будет работать по протоколу МЭК (в контекстном меню выбрать команду «Добавить»). Добавить МЭК в поддерживаемые протоколы.

Рисунок 7 – Создание устройства

Созданное устройство должно появиться в списке «Устройства сбора».

Рисунок 8 – Устройства сбора

После этого можно приступить к описанию дерева профиля для добавленного устройства.

Создаем в корне дерева новый узел – устройство. В свойствах узла выбираем устройство, которое создали в «Устройствах сбора». Протокол не выбираем. Тип, квалификатор, наименование – любые.

Рисунок 9 – Добавление профиля

Для созданного устройства создаем узел – протокол. Поле «Устройство» должно повторять родительское (на примере – «Сбор по МЭК870-5-101»). Назначаем протокол – МЭК (на примере – «IEC69870-5-101»). Тип, квалификатор, наименование – любые.

Рисунок 10 – Добавление протокола в профиль

Для созданного узла устройства создаем узел объекта, характеризующегося общим адресом ASDU. Таких объектов может быть несколько, соответственно создаем необходимое число таких узлов на текущем уровне иерархии. В поле протокол указываем – «Общий адрес ASDU». Поле «Устройство» должно повторять родительское. В качестве квалификатора указываем общий адрес ASDU, определенный в разделе «Необходимые знания» п.3. Тип, название – любые.

Рисунок 11 – Добавление объекта протокола

Для созданного узла общего адреса ASDU создаем узлы стандартных типов ASDU. Необходимые типы определяются разделом «Необходимые знания» п. 3. Количество стандартных типов ASDU для заданного общего адреса ASDU не ограничено. В поле протокол указываем – «Тип».  В качестве квалификатора выбираем идентификатор стандартного типа ASDU. Поле «Устройство» должно повторять родительское устройство. Тип, название – значения не имеют. На примере: ТИ передаются стандартным типом ASDU = 13 (значение измеряемой величины, короткий формат с плавающей запятой). Прочие поддерживаемые типы данных для протоколов IEC60870-5-101/104 перечислены в Приложение А.

Рисунок 12 – Добавление узла

Для созданного узла стандартного типа ASDU создаем узлы объектов информации (точек). В качестве квалификатора указываем адрес объекта информации. Адреса объектов информации определяется разделом «Необходимые знания» п. 3.

Количество объектов информации для заданного стандартного типа ASDU не ограничено. В поле протокол указываем – «Точка». Поле «Устройство» должно повторять родительское. Название значения не имеет. В качестве типа:

ü  для ТС указываем «Базовый булевый» или любой тип состояния. Опции параметра – «Мгновенные значения»;

ü  для ТИ указываем «Базовый аналоговый» или любой аналоговый тип (на примере – Ток). Опции параметра – «Мгновенные значения»;

ü  для ТУ указываем «Параметр телеуправления». Может быть задана опция «Телеуправление с предварительным выбором». Необходимость выбора перед ТУ согласуется в профиле (раздел «Необходимые знания» п. 3)

Рисунок 13 – Добавление точки измерений

В результате выполнения этих шагов имеем сконфигурированный профиль устройства, работающего по МЭК протоколу.

Рисунок 15 – Просмотр информации о протоколе

В первую очередь необходимо добавить описание нового измерителя с помощью следующих шагов:

1.    Выбрать в меню «Справочники» пункт «Протоколы». В открывшемся окне «Протоколы» проверить, что информация о протоколе МЭК внесена в список (как IEC60870-5-103). Здесь определяется структура адресации информационного объекта (точки).

2.    Выбрать в меню «Справочники» пункт «Устройства сбора». В открывшемся окне «Устройства сбора» создать устройство, которое будет работать по протоколу МЭК (в контекстном меню выбрать команду «Добавить»). Добавить IEC60870-5-103 в поддерживаемые протоколы.

Рисунок 16 – Создание устройства

Созданное устройство должно появиться в списке «Устройства сбора».

Рисунок 17 – Устройства сбора

Рисунок 18 – Добавление профиля

Для созданного устройства добавляется узел «Протокол». Поле «Устройство» должно повторять родительское (например, «Сбор по IEC69870-5-103»). Назначается протокол IEC60870-5-103. Тип, квалификатор и наименование заполняются согласно исходным данным.

Рисунок 19 – Добавление протокола в профиль

Для созданного узла устройства создаем узел объекта, характеризующегося общим адресом ASDU. Таких объектов может быть несколько, соответственно создаем необходимое число таких узлов на текущем уровне иерархии. В поле протокол указываем – «Общий адрес ASDU». Поле «Устройство» должно повторять родительское. В качестве квалификатора указываем общий адрес ASDU, определенный в разделе «Необходимые знания» п.3. Тип, название – любые.

Примечание. Для измерителей, работающих с протоколом IEC69870-5-103, значение «Общий адрес ASDU» обычно (но не обязательно) совпадает с физическим адресом измерителя.

Рисунок 20 – Добавление объекта протокола

Для созданного узла общего адреса ASDU создаем узлы стандартных типов ASDU. Количество стандартных типов ASDU для заданного общего адреса ASDU не ограничено.

В поле «Протокол» указываем тип элемента профиля, это может быть «Группа ТС» либо «Группа ТИ». Обратите внимание, что при выборе типа «Группа ТС» или «Группа ТИ» будет отличаться дальнейшая иерархическая структура уровней протокола.

Итак, в качестве квалификатора для узла «Группа ТС» или «Группа ТИ» необходимо выбрать идентификатор стандартного типа ASDU. Например, для «Группы ТС» квалификатор может иметь значение 1, 2, 20 (команда телеуправления), а для «Группа ТИ» – 3, 9. Поле «Устройство» должно повторять родительское устройство.  Тип, название – значения не имеют. На Рисунок 21 представлен следующий случай: ТС передаются стандартным типом ASDU = 1 (положение выключателя с меткой времени).

Рисунок 21 – Добавление узла

Для созданной группы ТС или ТИ необходимо установить номер функции и номер информации, эти параметры определены в стандарте IEC69870-5-103 либо в документации к используемому для опроса устройству. В поле «Протокол» устанавливается «Номер функции», тип – «Объект протокола». На примере устанавливается номер функции 11.

9.1 Передача параметров с двойным регистром

В реализованном протоколе Modbus предусмотрена передача значений с использованием двойного регистра, объединяя рядом стоящие регистры. Таким образом, становится возможным передавать большие значения и вещественные числа, не умещающиеся в стандартный размер регистра Modbus.

Двойной регистр состоит из двух обычных 16-ти разрядных регистров, составляющих вместе 32 разрядный ТИ. При этом возможны два варианта:

ü  старший 16-разрядный регистр Modbus имеет адрес, следующий за младшим (различающийся на единицу);

ü  младший 16-разрядный регистр Modbus имеет адрес, следующий за старшим регистром (различающийся на единицу).

11.1 Общие сведения

Технология OPC (Open Platform Communications) — представляет собой семейство программных технологий, предназначенных для управления объектами автоматизации и технологическими процессами.

OPC DA (Data Access) – спецификация программных интерфейсов для обмена данными в реальном времени с ПЛК, РСУ, ЧМИ, ЧМУ и другими устройствами.

OPC DA базируется на технологиях MS Windows: OLE, ActiveX, COM/DCOM.

Комплекс РСДУ поддерживает информационный обмен на основе спецификации OPC DA.

Для организации передачи данных из комплекса РСДУ во внешние системы на основе спецификации OPC DA необходимо выполнить описание профиля протокола с помощью приложения «Справочная информация по распределенной системе сбора» (DAView.exe) и настроить источники данных с помощью приложения Навигатор БД в разделе «Комплекс автоматики " Распределенная система сбора».

Настройки и запуск сервера OPC DA представлены в документе «Сервер передачи данных по OPC. Руководство администратора».

Рисунок 112 – Ввод параметров точки мониторинга

Примечание. Выделение ТС из битовой строки в протоколах MODBUS и IEC60870-5-101(104).

В протоколе IEC60870-5-101(104) при использовании типов данных 7 или 33 (строка из 32 битов) масштабирующий коэффициент для параметра ТС будет указывать на порядковый номер ТС, который должен быть выделен из битового массива. При использовании отрицательного коэффициента – значение бита будет инвертировано (то есть ноль будет интерпретироваться как единица, а единица – как ноль). Допустимыми значениями являются от -32 до -1 и от 1 до 32.

В протоколе MODBUS TCP (RTU) номер, прописанный в масштабирующем коэффициенте, будет также выделять бит в этой позиции и возвращать либо 0 (ОТКЛ), либо 1 (ВКЛ). При использовании отрицательного коэффициента – значение бита будет инвертировано (то есть ноль будет интерпретироваться как единица, а единица – как ноль). Допустимыми значениями являются от -32 до -1 и от 1 до 32.

После нажатия кнопки «Далее», на следующем шаге, необходимо выбрать какие архивы параметра требуется хранить в БД РСДУ5.

После нажатия кнопки «Готово» созданный сигнал будет отображен в списке параметров измерительного прибора.

14 Применение изменений в подсистеме сбора и передачи

Для того чтобы внесенные изменения вступили в силу, например, чтобы новые параметры начали обрабатываться, необходимо выполнить переинициализацию серверного модуля, ответственного за обработку измененных параметров.

В РСДУ5 реализация разных протоколов сбора и передачи представлена в разных серверных модулях, поэтому может потребоваться выполнить переинициализацию разных модулей.

14.1 Перезагрузка модуля «Коммуникационный сервер сбора и передачи»

Модуль «Коммуникационный сервер сбора и передачи» обрабатывает протоколы ION, IEC101, IEC104, СПЕ/СПГ/СПТ, ИРТ, Modbus TCP/RTU, ТЭКОН.

Для перезагрузки серверного модуля «Коммуникационный сервер сбора и передачи» (dcsd) на соответствующем хосте сбора необходимо выполнить следующие шаги:

1.    Подключиться к хосту сбора по сетевому протоколу SSH (например, с помощью клиентской программы Putty).

2.    Выполнить команду перезагрузки модуля сбора

ema reload dcsd

3.    С помощью клиента сигнальной системы CSignal убедиться, что модуль сбора успешно стартовал, о чем должно поступить сообщение в сигнальную систему РСДУ о загрузке модуля.

4.    Перейти на хосте сбора в директорию /var/log/ema и проверить лог-файл модуля dcsd на наличие ошибок. В лог-файле должны отсутствовать строки со словом Error. Допускается наличие предупреждений (строк со словом Warning).

5.    Если в лог-файле ошибок не обнаружено, то это может означать, что новый сигнал с прибора теперь будет обрабатываться в комплексе РСДУ. Для просмотра текущего значения нового параметра необходимо вывести данный параметр на панель (см. «Картридж “Настройка информационных панелей”. Руководство администратора»), либо воспользоваться клиентом коммуникационного сервера DCSClient.

14.2 Перезагрузка модуля «Сервис приема макетов 80020 АИИСКУЭ»

Модуль «Сервис приема макетов 80020 АИИСКУЭ» выполняет обработку макетов 80020 АИИСКУЭ.

Информация о перезагрузке сервиса представлена в документе «Сервис приема макетов 80020 АИИСКУЭ. Руководство администратора».
